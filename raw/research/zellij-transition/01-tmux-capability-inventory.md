# tmux capability inventory for the Zellij transition

**Status:** source-verified inventory, 2026-07-15

**Count:** 65 current capabilities/contracts/couplings

**Purpose:** migration ground truth for replacing tmux + SwiftTerm with a platform-neutral `SessionHost` backed by bundled Zellij, Electron, xterm.js, and node-pty.

## Scope and interpretation

This inventory applies the spike's non-negotiable product invariants: the real vendor TUI remains the pane content; Hive owns layout and focus; human input outranks automation; renderer activity never steals focus; a viewer/attach crash must not kill the session; and the packaged host must not depend on a user's multiplexer installation or configuration (`raw/bundled-zellij-terminal-stack-spike.md:35-43`). The minimum seam is `create`, `attach`, `writeAutomated`, `capture`, `inspect`, `terminate`, and `list`; its locator must carry tenant/Hive-home, agent, generation, and host address without tmux vocabulary or a shared default socket (`raw/bundled-zellij-terminal-stack-spike.md:92-113`).

“SessionHost equivalent” below means both the operation and the observable postcondition. An implementation that merely makes the analogous Zellij CLI call is not conformant when the current contract verifies state afterward.

Risk meanings: **high** can kill, misroute, overwrite human input, lose recovery, or leak paid processes; **medium** causes degraded diagnostics, UI behavior, portability, or operational control; **low** is localized and directly testable.

| A15 | **Viewer independence and crash reattachment.** Detached tmux owns the TUI; SwiftTerm owns only an attach client. | A16 | **Current host identity has no generation.** Agent database IDs are unique per holder, but `agentTmuxSession(name)` omits the agent UUID/generation and is reused when a name is reissued (`src/schemas/agent.ts:75-79`; `src/daemon/spawner-impl.ts:1922-1937`). ## B. Automated input, capture, delivery, and readiness (18)

| ID | Current capability / contract and implementation | Dependents | Platform-neutral `SessionHost` equivalent | Risk |
|---|---|---|---|---|
| B01 | **Arbitrary automated input travels on stdin, not argv.** Text is loaded into a random named tmux buffer via stdin, then pasted into the exact pane (`src/adapters/tmux.ts:207-229`). ## Known weaknesses and improvement opportunities

These are not optional capabilities to reproduce. They are current failure modes or brittle approximations that the new stack should replace while retaining the surrounding contract.

| Priority | Opportunity | Evidence and required improvement |
|---|---|---|
| 1 | **Make generation identity real.** | A16/B17: tmux name, prompt file, delivery lock, UI target, and liveness reuse `<agent>+<home>` even though the database has a unique agent UUID. Put UUID/generation in the neutral locator and reject stale operations. This also removes aliasing across respawns for terminal and worktree-derived readers. |
| 2 | **Unify initial and recovery readiness.** | B12–B16: initial spawn correctly uses screen redraw + actual process + hooks/artifacts; recovery still uses a fixed ten-second hook/artifact wait and screen only for errors. One conformance-tested readiness state machine should serve both. |
| 3 | **Replace timed paste with acknowledged automated input.** | B01–B07: stdin buffer delivery is safe for content, but correctness still depends on 500 ms and vendor turn events. Build an authenticated persistent control attachment/bridge that separates paste, key, submit, transport acceptance, and TUI acknowledgement. |
| 4 | **Make human ownership intrinsic, not a filesystem hint.** | B06/E06: marker files are crash-prone advisory state; Workspace startup clears them, and correctness depends on classifying every AppKit input path. Tie the lease to authenticated attach handles and generation, fail closed on uncertainty, and cover paste/IME/accessibility/programmatic input. |
| 5 | **Use OS containment instead of post-hoc `ps` ancestry.** | C02/C06/C07/C13: parent walks must race reparenting and do not carry a process start token. Unix process groups/cgroups and Windows Job Objects can make descendants and auxiliaries owned by construction and eliminate PID-reuse kills. |
| 6 | **Remove the external/user-configured multiplexer dependency.** | A01/A12/E03/E04: current code executes `tmux` through PATH/login shells and inherits tmux runtime conventions. Launch the checksummed bundled Zellij by absolute path with private runtime/socket/cache/config directories, as the spike requires. |
| 7 | **Separate renderer close from explicit instance stop.** | E12: current clean quit kills everything while the target invariant requires sessions to outlive the whole UI. Provide explicit Stop Hive/Terminate Agent actions; ordinary window/renderer close only detaches. |
| 8 | **Reattach automatically after host-generation replacement.** | E02/E03/E11: an existing agent attach client exits when its tmux session is killed for recovery/control restart; the pane view is not rebuilt merely because its feed `tmuxSession` field changes (`workspace/Sources/WorkspaceCore/ProjectState.swift:324-340`). AttachHandle should reconnect only when the daemon publishes the intended new generation, never by same-name polling. |
| 9 | **Persist direct provider artifact locators.** | D02–D06: exact session IDs now prevent the old path-alias bugs, but Codex lookup still recursively scans/sorts rollouts (and ordinary lookup caps candidates at 100). Persist the verified artifact path/identity when learned and validate it on read. |
| 10 | **Replace textual screen heuristics with structured host state where possible.** | A07/B10–B15: plain visible capture loses ANSI/style/scrollback and locale/version-specific error text can false-match. Keep current-screen capture for the real TUI, but use structured process exit/output revision/attach state for liveness and diagnostics. |
| 11 | **Consolidate the root launch path.** | Worker sessions use `TmuxAdapter.newSession` with prompt indirection, validation, diagnostic hold, history, timeout, and readback; root commands construct raw `tmux new-session` argv and embed the combined brief/memory index directly (`src/cli/orchestrator.ts:237-310`, `:328-395`). The root can therefore miss worker hardening, including the ~16 KB imsg workaround. Route both through SessionHost. |
| 12 | **Make multi-instance attachment authenticated and cross-platform.** | A01/A14/E03: current isolation relies on tmux Unix socket namespace and `/dev` client TTY inspection, with no neutral authenticated attach protocol. Use private endpoints plus tenant/session-generation authentication; expose attachment identity without Unix TTY strings. |
| 13 | **Define prompt lifecycle and remove the remaining ARG_MAX ceiling.** | A11: the workaround avoids tmux imsg but leaves a 0600 file keyed by reusable session name and expands its entire contents into one argv, so `ARG_MAX` remains the ultimate limit. Stream prompt bytes to the vendor/control bridge, make files generation-specific, and delete after verified handoff. |
| 14 | **Make capture dimensions/state explicit.** | B10/E05: readiness compares plain captures without recording terminal size; resize can look like activity. `capture`/`inspect` should return rows, cols, revision, alternate-screen state, and text so callers can distinguish resize redraw from ongoing output. |
| 15 | **Replace macOS-only monitoring.** | C04/C13: `vm_stat`, `ps` formats, Unix signals, `/bin/zsh`, `/dev` TTYs, and process groups do not satisfy Linux/Windows parity. The SessionHost conformance suite needs platform adapters and real runaway/escaped-child tests per OS. |

## Migration acceptance checklist derived from the inventory

The transition is functionality-preserving only when the same conformance suite runs against the current tmux host and bundled-Zellij host and proves:

1. Tenant + agent UUID + generation exactness across create, attach, write, capture, inspect, terminate, and list—including stale locators and same-name respawn.
2. A 100 KiB+ multiline prompt and automated message containing quotes, Unicode, shell metacharacters, control characters, and bracketed paste reach the intended TUI byte-for-byte without appearing in argv.
3. Human typing/paste acquires ownership before the first PTY byte; every automated priority defers; release is measured; output/status never focuses a pane.
4. Slow reasoning is alive, an idle prompt is alive via non-screen evidence, a frozen process is dead, and an animated wrapper with a dead vendor child is dead—for both spawn and resume.
5. Attach/viewer/xterm/Electron crashes leave the vendor TUI alive and reattachable; explicit agent/tenant termination captures auxiliaries and escaped descendants and returns no success with survivors.
6. Resource attribution catches a runaway grandchild, stopped/gone state is measured immediately, and process identity cannot kill a reused unrelated PID.
7. Clean UI close detaches only; explicit Stop Hive performs the verified all-agent-then-root shutdown and offline stale cleanup remains available.
8. Real Claude, Codex, and Grok TUIs preserve keyboard, mouse, alt-screen scroll, resize, focus, visible capture, exit/disconnect reporting, and at least four simultaneous panes without Zellij UI chrome.
9. Provider telemetry, live-model, Graphify-call, and recovery readers stay pinned to the exact provider conversation/artifact across worktree reuse and reject ambiguous fallback evidence.
10. The packaged binary is invoked by absolute path with Hive-private runtime/socket/cache/config and works with both `tmux` and `zellij` absent from `PATH`.
