Codex 0.144.4 hidden-bootstrap compatibility evidence
Captured: 2026-07-16

Evidence boundary
-----------------
This is local, version-specific CLI help and generated-schema evidence. It opened
no thread, sent no prompt, started no model turn, and spent no provider quota. It
does not claim live TUI presentation or app-server turn acceptance.

The current official Codex manual documents configuration layering, project
trust, one-run `-c` overrides, and `developer_instructions` in custom agent-role
configuration. It does not name the app-server `developerInstructions` field or
establish replacement semantics for `baseInstructions`. The app-server field
evidence below is empirical evidence from the installed 0.144.4 schema, not a
claim about public prose or later Codex versions.

Interpretation
--------------
The non-`--experimental` 0.144.4 bundle contains `developerInstructions` on both
thread start and resume, while initialize defaults `experimentalApi` to false.
Hive uses the start field for fresh read-only app-server workers. Hive's current
app-server manager has no durable thread/resume owner, so crash recovery remains
on the TUI resume path. The schema alone does not prove the instruction-role
semantics or that `baseInstructions` replaces built-in instructions; Hive makes
no such schema-derived claim and does not use `baseInstructions`.

Hive treats 0.144.4 as its own compatibility floor for this behavior. This is a
Hive product gate, not an OpenAI support-floor statement.
