A. Accessibility Inspector — hierarchy + audit
----------------------------------------------
1. Launch HiveWorkspace (or the Gate10-hosted test window if queen points you
   at a probe binary) with one live terminal pane attached to a sessiond
   shell or vendor TUI.
2. Open Accessibility Inspector → target the terminal content area.
3. Confirm against the machine dumps (ax-tree-*.txt):
   - role AXTextArea / textArea
   - parent/child: one row staticText element per visible terminal row
   - frames non-zero for non-empty rows when window is on-screen
   - focus follows first-responder
   - value / selectedText / insertion line match what is on screen
4. Run Inspector → Audit on the terminal element.
5. Capture the audit summary into:
     human-inspector-audit-transcript.txt
   Replace the PENDING_HUMAN placeholder. Include pass/fail counts and any
   finding that names HiveTerminalView / row elements.

Pass criteria:
  * role + child row count consistent with ax-tree-input.txt shape
  * no stale/duplicate row elements after scroll/resize/teardown
  * audit does not report broken parent/child for the terminal subtree

B. VoiceOver — navigate, type, select, scroll, lifecycle
--------------------------------------------------------
1. Enable VoiceOver. Focus a live Hive terminal pane.
2. Navigate by row (VO-Down/Up or equivalent) through several semantic rows.
3. Locate the cursor / insertion point announcement.
4. Type a short command; confirm committed output is announced or readable.
5. Select text (Shift-arrows or drag); confirm selection is spoken.
6. Scroll scrollback; confirm row focus survives without a full-tree wipe.
7. Force a reconnect or mark-lost path if available; confirm lifecycle label
   ("Terminal lost/exited/…") is spoken or inspectable.
8. Close the pane; confirm no hanging AX focus on destroyed rows.

Capture a short operator transcript (what you did + what VO said) into:
  human-voiceover-transcript.txt
If you can record screen+audio, drop a path reference in that file (do not
commit large media into the repo — keep media outside and cite the path).

Pass criteria:
  * rows, cursor, selection, typed input, and lifecycle are all reachable
  * teardown does not leave a zombie focused element
