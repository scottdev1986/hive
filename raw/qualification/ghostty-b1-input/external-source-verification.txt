Pinned behavioral audit result
- NSEvent+Extension.swift writes UInt32(NSEvent.keyCode) into
  ghostty_input_key_s.keycode. The pinned engine resolves that platform-native
  code through its macOS keycode table; converting it to ghostty_input_key_e
  at the Swift boundary would diverge from the pinned app. Gate 8 therefore
  retains the native code and proves the mapping end-to-end with distinct
  physical left/right modifier Kitty bytes.
- consumed_mods, unshifted_codepoint, translated modifiers, composing state,
  repeat/release actions, and text lifetime follow that same pinned source.
- Ghostty.App.swift proves runtime clipboard callbacks receive the surface
  config userdata, and that read completion is asynchronous through
  ghostty_surface_complete_clipboard_request. Config.zig fixes the explicit
  OSC policy values used by Hive: clipboard-read=deny and
  clipboard-write=deny.

The web documents were fetched live. In-tree Ghostty code was used only after
the pinned raw GitHub sources above were independently fetched; the three
sources also present in the reviewed vendor artifact were byte-compared.
