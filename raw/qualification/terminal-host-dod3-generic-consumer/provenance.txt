WHAT THIS CERTIFIES
The frozen boundary launches a session for a consumer that is not Hive and not an
agent, and honours a working directory that is not a repository:

- generic command, non-agent: executable /bin/sh with arguments
  ["-c", "test -f terminal-host-demo.cwd && exec /bin/cat"]. No agent, provider,
  or Hive identity appears anywhere in the create request.
- non-Hive-repo cwd: a freshly created /tmp/nhl-<random>/"generic command cwd 工作"
  directory - outside any git repository, and containing both a space and
  non-ASCII characters, which is qualification row B's "Unicode/spaces and a
  generic non-repository working directory succeed".
- cwd actually honoured, not merely accepted: the child's own first act is
  `test -f terminal-host-demo.cwd`, a marker the harness created ONLY inside that
  directory. If the boundary ignored workingDirectory the test resolves the
  marker nowhere else and the child exits non-zero instead of reaching /bin/cat,
  so a passing run cannot be explained by a cwd that was silently dropped.
- opaque, project-agnostic key: "tenant://foreign-project/../../opaque session ✓"
  - deliberately not a Hive-shaped identifier, and carrying traversal-looking and
  non-ASCII text the boundary must treat as opaque rather than interpret.
- complete environment: only PATH and TERM are supplied; nothing ambient is
  inherited, so the demo does not depend on the Hive shell that launched it.

WHAT THIS DOES NOT CERTIFY
- It is not a standalone third-party consumer PROGRAM. The stricter reading of
  DoD-3 - shipping a separate non-Hive binary built against the boundary - was
  considered and ruled out of M1 scope by queen on 2026-07-21; this artifact
  records the in-tree generic-consumer proof instead. The fork was deliberate.
- It says nothing about the A2 project-neutral real-sessiond adapter, which
  remains PENDING (see docs/contracts/terminal-host-v1.md status).
- The Zig proof prints nothing on success, so the capture's evidence is its
  echoed exit code under `set -eu`, not per-assertion output. The assertions
  named above live in the cited code, and each fails the run if violated.
