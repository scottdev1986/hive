M1-B2 B2.1b exact close/quit lifecycle freeze evidence

Lifecycle behavior proved:
- Workspace pane close carries its held Hive SessionLocator through the CLI request.
- The daemon compares the complete locator before teardown. A stale generation returns
  session-locator-mismatch, leaves the current agent live, and invokes no kill; the exact
  locator positive control terminates that same agent.
- hive stop partitions tmux and sessiond records. Every live sessiond record is attempted
  through the authenticated daemon kill path before SIGTERM. Any unverified teardown keeps
  the daemon live and makes the stop command fail visibly.
- Daemon sessiond teardown captures the inspected host root before termination, requires the
  adapter's terminated/no-survivor result, and reaps plus re-reads the captured tree.
- AppKit returns terminateLater until hive stop exits zero. Failure replies false and presents
  the actual stop diagnostic; no deadline reports a successful quit while teardown is pending.
- A malformed present sessionLocator becomes an immediate feed schema error and never a
  partial agent snapshot.

Live native proof:
- A real sessiond host/provider pair was observed live by exact PID/start token.
- The production stopHive ordering invoked stopSessiondAgentSession, recorded a termination
  audit, proved both exact identities absent, and only then allowed the daemon signal.
- A fresh second sessiond generation was then created. Killing the Workspace publisher proved
  both exact identities absent through the independent 15-second crash-expiry backstop.
