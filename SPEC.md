## The vision

**Hive is building a custom terminal so that Hive can be integrated *into* that terminal.**

Every other orchestrator watches its agents from outside — through hooks that fire when a vendor chooses to fire them, through transcripts written after the fact, through screen scrapes that guess. Hive owns the terminal the agent runs in, so it can see what is actually on the wire.

The point of owning the terminal is **omniscience about what is happening in a TUI**:

- If the agent is **asking a question**, Hive knows.
- If the agent has **stopped working when it should be working**, Hive knows.
- If the agent has hit a **vendor cybersecurity or safety stop**, Hive knows.

And in each case Hive gets the user's attention so a human can handle it. An agent blocked behind a question nobody sees is an agent that has silently stopped; that failure is the reason the terminal is being rebuilt rather than adapted.

The second half of the vision is **the agent factory**. M2 rebuilds agent spawning and agents themselves into a factory, so that adding a new TUI — Kimi Code, opencode, whatever ships next quarter — is exactly two steps: *do the research, build the factory entry*. Not a new pipeline, not a new adapter subsystem, not a per-vendor fork of the spawn path. Vendor neutrality stops being a claim about how many vendors we happen to support and becomes a property of the spine.

Everything in the milestone plan below serves those two things: a terminal Hive can see through, and a factory that makes seeing through it cheap for every new agent.

## What this is

You `cd` into a project and type `hive claude`, `hive codex`, or `hive grok`. The Workspace app opens with the selected AI orchestrator — **queen** — in its master terminal pane, scoped to that folder. You talk to it like a tech lead: "build this feature," "figure out why the tests are flaky," "have Codex take a second look at the auth code." It doesn't write the code itself. It decomposes the work, spawns agents — Claude Code here, Codex there, big model for the hard part, cheap model for the changelog — and each agent appears in a terminal pane where you can watch it work and type into it. Agents talk to each other when they need to, regardless of vendor. When the work is done, it gets merged, and the panes go away.

What separates this from a demo is the safety and control stack: **quota-aware routing that durably reserves subscription-CLI capacity before spawning, durable peer delivery with critical controls that mechanically revoke authority rather than politely request compliance, an orchestrator that physically cannot write, and a rebase→verify→fast-forward-only landing protocol that makes "the tests and the typechecker passed on exactly what main now is" structural.** Worktrees, terminals, sandboxes, panes are table stakes we build because we must; the control plane over cross-vendor agents is the product — and, from this rebuild forward, the terminal is part of that control plane rather than a window we borrowed.

The strategic read is unchanged and stated honestly: several parties could close this gap — a vendor shipping cross-vendor teams, or an existing orchestrator growing a real control plane. The defensible ground is the safety stack plus vendor neutrality, which is why the agent factory (M2) is a milestone rather than a refactor. macOS only, by choice, for now.

## Where the rebuild stands

Hive is mid-rebuild. The product described above runs today on a tmux + SwiftTerm substrate that is being replaced, on `main`, progressively — not on a long-lived integration branch. Development is coordinated by a checksummed, pinned **old** Hive release with its own `HIVE_HOME`, database, sockets, and ports, so the tool building the rebuild cannot see the rebuild's runtime state (`M1-BOOT`, closed; `planning/backlog-outline.md` M1 Prerequisite).

Ground truth for board state, landed artifacts, and where the tracker lags reality is `planning/2026-07-20-board-planning-repo-reconciliation.md` §1–§4. Its standing observation is worth repeating here because it changes how the milestones below should be read: **the board understates engineering progress** (columns lag landings) while the definition-of-done verdict holds exactly — the atomic cut has not started, and zero of six human evidence gates are met. The milestone boundaries below are stated in terms of *exit criteria*, never column positions.

Two standing rules govern every milestone:

- **Qualify, don't presume.** `sessiond` (Zig), `HiveTerminalKit` (Swift), the vendored Ghostty, and the in-tree conformance-test ids exist, but in-tree code is a *candidate*, never evidence. Each story qualifies its artifact against external conformance sources and live matrices and keeps it only if it passes; rewrite is the fallback, not the default (user ruling Q4).
- **Gut, then rebuild.** No legacy shims, no dual-host canary, no renderer flag, no compat writes. The replacement proves itself on the full live vendor matrix first; then the old path is deleted atomically.

## Milestones

### M1 — Hive works using ONLY the new terminal

M1 is done when Hive runs on the new terminal alone: `sessiond` plus `HiveTerminalKit`, with tmux and every temporary harness gone — **the orchestrator session included, not exempted.**

The foundation M1 is qualifying is recorded in [ADR 0001 — macOS native terminal foundation](docs/adr/0001-native-terminal-foundation.md) (Decision, Consequences) and frozen behind the neutral [terminal host contract v1](docs/contracts/terminal-host-v1.md) (Required behavior §§1–10, Freeze qualification A–K). The control-side interface is `SessionHost`, which contains no tmux and no renderer vocabulary; all Hive policy lives above that seam and syscalls are implementation options, never API vocabulary.

**Exit criteria.** All of the following, none softened:

1. **The atomic tmux cut** — `#1` (STORY-001, gut all tmux terminal code) and `#2` (STORY-002, remove the SwiftTerm/tmux-attach agent-TUI hosting path) execute as **one atomic merge train** at the Removal Gate, with the full live vendor matrix re-run on the post-deletion tree. **The orchestrator session is included in the cut.** Pre-cut drain is performed by the pinned bootstrap build.
2. **`make terminal` and all testing-only support code removed** — `#59`. Every temporary harness or support path that exists only to exercise the terminal during development is deleted before M1 closes. An artifact that still ships or depends on a testing-only support path has not proven what M1 claims.
3. **The exit-command acceptance criterion** — `#60`. An agent's own exit command closes the *agent*; the **terminal pane STAYS OPEN**; and Hive records the closure and runs work preservation. Agent lifetime and pane lifetime are separate facts, and this is the first place M1 proves it.
4. **C2 packaging** — `#11`, per `planning/story-m1-c2-packaging.md`. A signed, notarized, self-contained universal artifact installs and runs the full M1 spine. Per the user rulings recorded in that doc: signing/notarization evidence is a **green `release.yml` run on the post-cut tree**, not a fresh per-cut attestation bundle; and **"clean machine" means a tmux-absence check only** — any machine where `tmux` is verifiably absent from `PATH`, measured with a negative control, not a fresh user account and not separate hardware.
5. **B3 smoke re-run post-cut** — `#9`. The replacement smoke harness on the sessiond/HiveTerminalKit spine has landed; it must re-run green on the post-cut tree.
6. **The deferred live-proof matrix cells** — `#36`. Cells J/K/I/G-live were split out of B1 by queen's direction (2026-07-17) and are not duplicate scope; M1 does not exit with them open.
7. **The human evidence gates.** The aesthetic bar is signed off by **the USER personally** against reference terminals (Ghostty app, Terminal.app, iTerm2) — a hard gate, no engineer proxy (user ruling Q5). The remaining deferred human evidence is tracked in `#45`.

The engineering tracks that reach those criteria — A0 contract freeze, A1 sessiond qualification, A2 the production `SessionHost` backend, A3 input arbiter, A4 close/reconnect/containment, B1 GhosttyKit qualification, B2 pane wiring, B3 smoke, C1 the beautiful blank terminal, C2 packaging — and their dependency edges are specified in `planning/backlog-outline.md` M1 and the per-story `planning/story-m1-*.md` docs. SPEC does not restate them.

### M2 — the agent factory

M2 is the factory. Adding a new TUI becomes: do the research, build the factory entry.

**Exit criteria:**

1. **An agent-agnostic spawn spine** — `#38`. One vendor-neutral spawn and registration path, not a per-vendor fork. **`#12` (Claude Code), `#13` (Codex), and `#14` (Grok Build) are proof targets under that spine, not independent pipelines** — the per-vendor S2.1/S2.2 structure in `planning/backlog-outline.md` M2 is superseded by `#38` (banner landed 2026-07-18; the banner is on the outline itself).
2. **Spawning is native to our terminals.** No tmux paste-based prompt delivery, no terminal-emulator borrowing: an agent is launched into a sessiond generation with an explicit env/cwd/argv contract, and sensitive payloads ride `0600` files rather than argv.
3. **Kimi Code and opencode are FIRST-CLASS agents** — `#63`. Native spawn, normal **and** yolo approval postures, protocol, and landing — held to **the same bar as Claude, Codex, and Grok**, not a reduced one. This is the criterion that proves the factory is a factory: if adding them requires anything beyond research plus a factory entry, M2 is not done.
4. **Status pipeline v2** — `#15`. StatusEnvelope v2 with source/freshness/confidence, and a live agent demonstrating **every current status promise** — working, idle, awaiting-approval, paused, stuck, done, failed, unknown — observed end to end. `#15` proves the **promises only**; it does not own new status semantics. Terminal pixels are never status truth (invariant I6).
5. **Message delivery over the new spine** — `#16`. normal / steer / urgent / critical delivered through the sessiond arbiter with **measured** receipt, and truthful per-vendor degradation stated rather than papered over.
6. **A vendor-TUI conformance corpus** — `#17`. Alt-screen, kitty keyboard, mouse reports, bracketed paste, OSC 52 — run per vendor inside the new terminal.

The belief-injection research folded into the outline's S2.2 (per-vendor flags, the three-way approval matrix, the nonce-based silence proof) remains valid *as research*; it is the content a factory entry is built from, not the structure M2 builds. Read `planning/backlog-outline.md` M2 S2.2 as sourced vendor detail under `#38`'s spine.

### M3 — communication

M3 is where the vision's omniscience half lands.

**Exit criteria:**

1. **TUI omniscience** — `#61`. The three detection classes, with user-attention escalation: the agent is **asking a question**; the agent has **stopped working when it should be working**; the agent has hit a **vendor cybersecurity/safety stop**. Detection alone is not the criterion — getting the user's attention is.
2. **Queen can open and close BLANK terminals** — `#62`. Terminal lifecycle is decoupled from agent lifecycle: a terminal is a thing queen can create and destroy with no agent in it, and an agent ending does not end its terminal (the other side of M1's `#60`).
3. **The durable-communication ladder** — `#18`–`#22`, staged on `docs/design/hive-communication.html`'s C-ladder: C0A durable core (v2 envelopes, pre-spawn identity reservation, digests/causal links/idempotency, content object store, WorkManifest + checksummed journal outside worktrees); C0B bounded truthful wakes (one delivery lane, provider-observed vs applied split, wake budgets, inbox cursors, explicit acks); C0C stranding recovery; C1A hierarchy-aware routes (lands with M4's schema); C1B cutover, which **deletes** the ambiguous legacy paths rather than shimming them.

### M4 and M5

M4 is the agentic hierarchy (`#23`–`#28`), staged on `docs/design/agentic-hierarchy.html`'s H-ladder: shadow records → direct+flat control plane → optional lead tier → promotion trains → recovery and succession → frozen stable projections. `hive_land` loses its fixed target and becomes `PromotionGrant`-derived at H1.

M5 is a **UI-readiness gate, not UI work** (`#29`–`#33`): every Split Horizon feature-ledger row has its underlying truth built and live-proven through projections, CLI, and typed operations, demonstrated with no new UI. Split Horizon then starts as pure presentation. The final story is the end-state demo: the dev build opens on an arbitrary non-Hive project, any vendor boots as queen, and the full loop — spawn → work → status → message → land — is proven.

Detail for both lives in `planning/backlog-outline.md` M4/M5 and the two design docs. M5's row families are named there but the rows are not yet enumerated; that refinement is deliberately deferred until M4-H5 freezes the projections.

## The architecture, in outline

Five things, and everything else is detail beneath one of them:

- **The daemon** (Bun/TypeScript) is the control authority: message bus, status table, approval router, memory store, run history, routing and quota ledger, resource watchdog. Agents speak to it over MCP. It owns lifecycle *intent*, authorization, capability epochs, status fusion, messages, and attention — and, per [ADR 0001](docs/adr/0001-native-terminal-foundation.md), it does **not** own terminal bytes, a PTY descriptor, a terminal grid, focus, or a renderer.
- **`sessiond`** (Zig, bundled universal executable) is the terminal substrate: a per-`HIVE_HOME` broker plus one isolated host per terminal generation, owning the PTY master, ordered input, canonical headless VT state, output sequence, journal, checkpoints, attached clients, and process evidence. Its boundary is the [terminal host contract v1](docs/contracts/terminal-host-v1.md).
- **`HiveTerminalKit`** (Swift/AppKit) owns presentation: the Ghostty Metal surface, render scheduling, first responder, key/text/IME/mouse encoding, selection, search, copy, accessibility, theme projection, attach state. Ghostty is adopted as an internal engine behind a pinned, fenced bridge — never as the Ghostty application, window model, or product identity ([ADR 0002](docs/adr/0002-ghostty-patch-budget-and-threat-model.md), [ghostty fork contract v1](docs/terminal/ghostty-fork-contract-v1.md)).
- **queen** is a read-only classifier: it decomposes, scopes, tiers, and spawns; it never touches files.
- **Agents** are the vendors' real interactive TUIs, sandboxed and worktree-isolated, recycled at the context quality line via handoffs and resumed with their conversation intact when a crash — not a decision — ended their process.

## Decisions

The fifteen numbered decisions below keep their original numbers. Each states the ruling and points at where the mechanism is specified; none is restated in full here. A decision marked **carried forward** was true in the prior SPEC and remains true. Where a decision's *mechanism* is being replaced by the rebuild, that is said in place; where a decision has been *superseded outright*, it appears in [Appendix A](#appendix-a--superseded-decisions) instead.

**1. How agents talk to each other** *(carried forward; transport being replaced)*. Both a place for messages to live and a way to wake the recipient. The place is the daemon's SQLite store, exposed to every vendor over MCP — the one API surface identical across vendors — with names durably reserved at the synchronous start of spawn so spawn and send cannot race. Messages carry explicit **priority and intent** rather than prose the model must interpret: `normal` waits for a turn boundary, `steer` is non-destructive mid-turn guidance, `urgent` cancels the in-flight turn, and `critical` atomically advances the capability epoch, revokes landing, denies stale approvals, and restarts the agent read-only in the same worktree. Every message carries a monotonic recipient sequence and the durable states `queued`, `injected`, `agent-acknowledged`, `applied` — **persistence is never reported as delivery, and injection is never reported as receipt.** No incoming delivery may own a human's composer: a synchronous typing lease blocks every submission path until the human's draft leaves the field. queen is a reserved destination, absent from the agents table, and its private supervisor restarts a dead root only when a daemon read positively proves live agents remain — an unreadable daemon is *unknown*, never permission. **What the rebuild changes:** the tmux paste-and-submit transport is replaced by delivery through the sessiond arbiter with measured receipt (M2, `#16`), and the durable core is rebuilt on v2 envelopes in M3 (`#18`–`#22`). The message semantics above survive that change; only the wire beneath them moves. Detail: `docs/design/hive-communication.html`, [daemon authorization](docs/daemon/authorization.md).

**2. How queen knows anything** *(carried forward)*. **Read what the tool measures and hands you; infer only what nothing tells you.** This is the governing rule for every sensor in the system, and it was learned expensively: Hive once divided a transcript's token count by a hardcoded 200,000 window, so agents at ~22% of a 1M window reported 100% full, and queen spawned fresh agents all day against a quota pool that reached 99%. **The cost of a wrong telemetry number is never a wrong display — it is every decision downstream of it.** Consequently **unknown is a first-class value and never a flattering default**: a measurement Hive does not have is reported as unknown, not 0, not 100, not a plausible constant, and a consumer must read unknown as "do not act." Proof of life measures the agent, not the screen: a redraw counts only while the binary Hive launched is still alive in that pane's process tree, and there is deliberately **no wall-clock spawn deadline**, because a fixed timer measures *acting* and a high-effort model thinks before it acts. Agents get **human first names** from a curated pool; a name has exactly one live holder and closure is durable. queen is idle by default — it polls nothing and never scrapes a terminal. **What the rebuild changes:** the `capture-pane` heartbeat disappears with tmux; status truth is re-proven end to end on the new spine in M2 (`#15`), and terminal pixels are never status truth (invariant I6). The *sources* of omniscience get better in M3 (`#61`), not looser. Detail: [orchestrator status](docs/daemon/orchestrator-status.md).

**3. How agents avoid destroying each other's work** *(carried forward)*. **Worktree per writing agent** — own branch, own checkout, shared object store; readers share the main checkout. Isolation is the floor; the strategy is planning, and queen lands shared interfaces before fanning out and gives every agent an explicit file scope. **Every writer lands its own finished work**, because completed work stranded on a branch is indistinguishable from work never done: commit, `git rebase main`, re-run the tests **and** the typechecker on the rebased branch, then `hive_land` with the capability epoch issued at spawn, which the daemon verifies before `git merge --ff-only`. Fast-forward-only is the whole trick for concurrency — the loser of a race is simply rejected and re-rebases. Tests *and* typecheck because a suite is green on trees the compiler rejects; two agents each added a version module, each branch passed its own tests, and the merged main failed `tsc` on a symbol collision no test either could have written would have caught. A rebase *conflict* means planning failed: abort and escalate, never force. A refused landing must **name the specific blocker** and must never guess — Hive will not rebase for a writer and will never discard uncommitted changes it did not write. **Work is either merged, intentionally preserved, or explicitly surfaced, never silently lost**: a killed agent holding unmerged commits has its branch preserved to `refs/hive-preserved/<branch>`, and killing is a kill of the agent's *processes*, not merely its session ([agent teardown](docs/daemon/agent-teardown.md)). The same guarantee covers messages: "injected" is a promise with a deadline, swept and surfaced in one line, and the sweep triages busy-versus-deaf before it speaks so it does not train its only reader to ignore it.

**4. Who answers the permission prompts** *(carried forward)*. Full autonomy for writers, **one deliberate setting away, and sandboxed out of the box**. `autonomy = "sandboxed"` is the shipped default; `dangerous` is one click in the Workspace menu and persists through the daemon's operator-gated endpoint — agent credentials cannot touch it, so no writer can raise its own autonomy. Getting to the first turn also means answering the vendor's own blocking dialogs in advance, scoped to the agent's worktree and never to the user's repository root; whatever cannot be scoped to a Hive worktree, Hive does not touch. Two boundaries survive every flip of the dial, mechanically: autonomy grants write authority only to **writers** — a reader keeps its deny list or read-only sandbox, and the read-only replacement a critical control spawns cannot regain shell access through a config default. The `hive_land` credential is one-shot, with exactly one deduplicated pending approval and a queen-granted re-arm rather than a dead end. Detail: [launch mechanics](docs/providers/launch-mechanics.md), [daemon authorization](docs/daemon/authorization.md). The three-way approval matrix each vendor must build and live-prove — manual, sandboxed-autonomous, unsafe-bypass — is an M2 factory-entry requirement (`planning/backlog-outline.md` M2 S2.2, under `#38`).

**5. What "memory" actually means** *(carried forward)*. "Memory" was three things wearing one name: **conventions** (a committed instructions file the *vendor* loads natively — Hive neither picks the filename nor injects the contents), **run state** (the daemon's SQLite, which dies with the session, and calling it memory was the conflation to kill), and **durable knowledge** (narrative lessons compiled into owned articles). Durable knowledge follows a compiler, not a notebook: immutable `raw/` observations compile into `wiki/` articles Hive owns and rewrites, in per-repo and global scopes, so a wrong fact and its correction can never coexist with no machine-readable winner. A lesson must be **durable, non-derivable, and load-bearing**. Provenance and verification status are first-class, and **an article naming a concrete path, command, or flag is re-checked against the repo before it drives an action**. Agents receive the **compiled index only** — capped, newest-first — with article bodies pulled on demand, because a small injected surface is an accuracy budget and not merely a cost budget. Detail: [agent memory](docs/agents/memory.md).

**6. Who picks the model** *(carried forward, with its manifest superseded)*. **queen classifies; a table resolves.** The judgment call — this subtask is `deep`/`standard`/`cheap` — is a thing LLMs are genuinely good at and never goes stale; which model that means is a lookup, because an LLM's pricing and capability knowledge froze at its training cutoff. Routes must not assume account entitlements. The binary now ships with **no model knowledge at all** — no compiled table, no manifest, no alias constants; routes derive per cell from live discovery, the user's `routing.toml` (which always wins), and a last-known-good snapshot, and a cell none of those can author **refuses the spawn** rather than guessing. Model *judgment* belongs to external benchmarks consumed as data and to the user's placements, never to the binary. An **execution identity is an intention, not an observation**: `model` is the immutable launch identity a control restart must reproduce, `live_model` is what the transcript says is actually running, and conflating them made Hive account quota against models nobody was running. Routing is **quota-aware**: capacity is read from the provider's own program-readable surfaces at every start, denominated in percent because a provider reports the *fraction* of a window consumed and never the window's size, atomically reserved before any worktree or process exists, and reconciled in **both** directions — a provider reading that frees capacity is believed, because the provider measured the account and Hive only estimated its share of it. Capacity and money are separate gates, and Hive never spends a user's finite reset grant to get its own way. Detail: [routing policy](docs/routing/routing-policy.md), [rejected approaches](docs/routing/rejected-approaches.md), [quota and headroom](docs/routing/quota-and-headroom.md), [provider quota surfaces](docs/providers/quota-surfaces.md), [capability discovery](docs/providers/capability-discovery.md).

**7. What happens when a context fills up** *(carried forward)*. Quality dies long before the window fills — but **not at a threshold and not at a percentage**. Degradation onset is an absolute token count with a large model-specific constant, so window size predicts almost nothing: a 1M-window model can be at 40% success by 32K while a 200K-window model still holds 84% at the same absolute depth. The real trigger is **error state, not depth** — self-conditioning on one's own prior mistakes is the best-evidenced predictor of the next one. So: retire on **task completion** (normal, free), on **repeated failure at the same sub-goal** (quality), and only then on an absolute per-model token ceiling (backstop). **Admit** and **retire** are two lines, not one, and the gap between them is the room an agent needs to finish what it accepted. A handoff carries little and only what the repo cannot say: a computed half extracted from the worktree that cannot hallucinate, a short written half, and above both a **pinned block** of the user's constraints re-injected byte-identical and never passed through a summarizer. Recycling is not the safe direction to err — believing it was is what justified the 200K bug. Detail: [context and recycling](docs/agents/context-and-recycling.md).

**8. Skills, and when to build one** *(carried forward)*. One `SKILL.md` folder serves every agent regardless of vendor, stored per-repo in `.hive/skills/` and globally in `~/.hive/skills/` and linked into each tool's native discovery path at spawn. **The directory decides what ships**: `skills/` is compiled into the binary, while `.hive/skills/` and `.claude/skills/` are Hive's own development kit and nobody else's business — enforced by a test that reads the built binary's bytes, not by a promise. The installer never destroys a file a human owns: absent it writes, identical it leaves, **different the human's copy wins** and the drift is reported. Detection is a **rule of three** over the daemon's task history — same task shape ~3 times, propose a skill. Propose, not create: silent accumulation is how you get sprawl nobody audits.

**9. Terminals are sessions; panes are viewers** *(carried forward in principle, rebuilt in mechanism)*. The original framing — the agent lives in the window — hides a failure mode: closing david's window kills david mid-task. The inversion survives the rebuild intact, but its substrate changes. Agents lived in daemon-owned tmux sessions; they now live in `sessiond` generations, each owning exactly one vendor process generation and its PTY, with Workspace panes as attached clients under a visibility lease. Closing a viewer detaches; the agent keeps working; reattaching reopens the lens. Two rules tighten in the rebuild: every operation addresses an exact instance, subject, and generation, and a stale or ambiguous locator **fails with typed evidence** rather than acting on the wrong target; and **user close means verified termination, never background detach**. The lifecycle decoupling this decision implies is finished in `#60` (an agent's exit leaves its pane open) and `#62` (queen opens and closes blank terminals). Detail: [ADR 0001](docs/adr/0001-native-terminal-foundation.md), [terminal host contract v1](docs/contracts/terminal-host-v1.md), [visibility contract v1](docs/contracts/terminal-host-visibility-v1.md).

**10. Layout is the Workspace's, not the window manager's** *(superseded in mechanism — see [Appendix A](#appendix-a--superseded-decisions))*. The principle survives: Hive owns its own geometry, because cheap viewers pile up and macOS cascades them into a deck of cards with the one window that matters buried underneath. The osascript wall of native terminal windows is replaced by the native Workspace's own pane topology. Detail: [Workspace blueprint](docs/workspace/blueprint.md), [UI design system](docs/workspace/ui-design-system.md).

**11. queen physically cannot code** *(carried forward)*. An orchestrator that helpfully edits files is an unsandboxed writer in the main checkout, burning its own context on diffs, outside the approval flow. Both CLIs *will* drift into coding if merely asked nicely — a system prompt is a suggestion, permissions are a constraint. So it is mechanical: read-only sandbox or deny rules on the write tools plus read-only Bash. Its only actuators are the Hive MCP tools. Reading is encouraged; writing is impossible. Every root-only behavior is launch-scoped and delivered from Hive's own home, so a crash cannot leave an ordinary launch in that repo inheriting Hive. **No small-edit carve-outs** — carve-outs are how invariants erode.

**12. The machine outlives any one agent** *(carried forward)*. Hive runs other people's code on your only computer, and amplifies every individually-reasonable mistake by running many agents at once. One agent's unit test faked a long-poll with an instantly-resolving mock, turned a pump loop into a ~1.5 GB/s allocator, produced no output, got retried, and stacked to ~164 GB until macOS jetsammed the machine and killed every agent, the daemon, and the user's other work. The lesson is not "fix that test" — agents will write that test again — it is that **Hive must degrade before the machine does**. So the maintenance tick is also a resource sweep: per-process memory ceilings under Hive-owned sessions, a spawn pause under system memory pressure, orphan reaping, and history pruning. Every kill and pause lands as a durable message to queen, so degradation is visible and attributable, never silent. Trusting queen to notice was rejected on arithmetic: the machine died about four minutes after the first runaway appeared.

**13. What survives a crash** *(carried forward)*. **A process is disposable, a worktree is durable, and the conversation inside the tool is the expensive thing in between** — so Hive must be able to put a process back around a surviving conversation. Recovery is classify-and-recover on the reconciliation tick and at daemon startup, because after a machine crash the daemon restart *is* the recovery moment and requiring a human step reproduces the stuck-forever failure. Resume is gated on **proven identity**: a vendor whose session metadata carries no independent binding to Hive's process incarnation is fail-closed rather than resumed on a guess. A resumed conversation is not a checkpointed process — the recovery notice tells the agent to re-verify its in-flight edits rather than pretending continuity — and the retry cap is what keeps automatic resume from becoming a crash-loop amplifier. Detail: [database resilience](docs/daemon/database-resilience.md). **Known gap:** recovery currently misclassifies audited kills as crashes (`#66`) — see [known integrity gaps](#known-integrity-gaps).

**14. Starting into a repo** *(carried forward)*. Hive works with any repo; nothing Hive-repo-specific ships in the product. That forbids compiling in a doc name, a design-doc convention, or a build command. The answer is **repo-neutral doc discovery on demand**, not a stored model of the repo: a walk of the tree when a spawn needs it, ranking the primary design doc by inbound citations, spending zero model tokens and caching nothing, so a task citing "DESIGN §3" in a repo whose primary is `DESIGN.md` gets the same scoped extraction Hive's own repo gets for "SPEC §6". Conventions reach agents through the *vendor's* native instructions file, never through Hive machinery, and Hive does not detect a repo's build or test commands — the landing gate says "re-run the tests" and "typecheck it" in repo-neutral wording, because a guessed command an agent runs is worse than a rule it applies. Doc discovery is the *only* thing Hive derives from a repo it did not write. Detail: [briefing](docs/agents/briefing.md).

**15. The shape of the whole thing.** The daemon is the control authority and the keystone; `sessiond` is the substrate where agents live and `HiveTerminalKit` is the surface where they are seen; repo-neutral briefing is the portability seam; queen is a read-only classifier; agents are sandboxed, worktree-isolated, disposable, and resumable. The rebuild does not change that shape — it replaces the two borrowed pieces (tmux, SwiftTerm) with pieces Hive owns, which is what makes the vision's omniscience possible at all.

## Tech stack

One language for the control plane: **TypeScript on Bun, compiled to a single binary.** The decisive constraint is MCP — the daemon *is* an MCP server and the official TypeScript SDK is the reference implementation. Bun buys fast cold start, built-in SQLite (`bun:sqlite`), and `bun build --compile` for one distributable binary with no runtime to install. Go is the right shape and would win if the MCP TS SDK did not exist; Rust pays systems-language tax on a scripting-language problem; Python has a miserable CLI distribution story.

The rebuild adds two languages where they are load-bearing rather than convenient: **Zig** for `hive-sessiond` (PTY, VT state, journal, checkpoints — a systems problem that genuinely is one) and **Swift/AppKit** for `HiveTerminalKit` and the Workspace (Metal surface, IME, accessibility — a platform problem that must be native). Everything else is process-glue and JSON.

The concrete choices: Streamable HTTP on localhost for the daemon's MCP transport (one daemon, one endpoint, N agents); SQLite via `bun:sqlite` for inboxes, status, run history, and durable token events; `Bun.spawn` for git and platform tools, with no wrapper libraries because the CLIs are the API; the hive binary itself as hook glue; on-demand daemon spawn with a pidfile; Zod schemas shared across messages, handoffs, and routing; `bun test` for the suite, with the interesting coverage being live integration tests.

Distribution: immutable version directories activated by an atomic symlink, one push to `main` is exactly one patch release, and no version number is ever typed by a human. Because a Unix process keeps executing its open image after the symlink moves, **an updated binary must never adopt the daemon it just replaced** — the handshake's content-addressed build hash refuses it. See [versioning and release](docs/release/versioning-and-release.md), [distribution](docs/release/distribution.md), [update experience](docs/release/update-experience.md), [acceptance testing](docs/release/acceptance-testing.md).

One deliberate boundary: platform adapters and vendor adapters live behind small interfaces. Those are the only places vendor and platform specifics are allowed, and they are exactly where the roadmap grows — which, restated in M2's terms, is the agent factory.

## Known integrity gaps

The M3 communication work exists partly because these classes of silent failure are real and **measured**, not theoretical. On 2026-07-20 three agents — maya, david, and john — died through authorized, audited operator kills that nobody intended, and the investigation that defines the defect is `planning/2026-07-20-operator-kill-defect.md` (investigation only; it fixes nothing). The family it opened:

- **`#64`** — the Workspace pane-close path kills agents without user intent (`.closePane` double-dispatch; the pane is retained after the kill). Closing a viewer is supposed to detach, per decision 9. Here it killed.
- **`#65`** — `hive stop` kills all agents before verifying that shutdown may proceed, leaving a live app and daemon with a dead fleet.
- **`#66`** — recovery misclassifies audited kills as crashes: `closedAt` is written after teardown, and resume downgrades sessiond → tmux. This is decision 13 failing in the direction that matters, and it is why "a reap message is not a completion report."
- **`#67`** — `hive_send` refuses live idle agents with a mismatched-`SessionLocator` error while the database, the bindings, and the processes all agree.

They belong in this document rather than only on the board because they are the same shape as the vision's third bullet: **something that mattered happened, and nobody was told.** A kill nobody ordered, a fleet nobody was told died, a message refused for a reason that is not true. Detection classes in `#61` and the durable core in `#18`–`#22` are what make these observable rather than reconstructible after the fact — and until they land, every negative here needs a positive control before it is believed.

## Open questions and deferred bets

Named honestly, not papered over:

- **Self-land versus a review gate.** Every writer lands its own finished work through rebase → retest → ff-only (decision 3), and industry practice leans hard the other way. Green tests and a clean typecheck on exactly the tree main becomes is real, but it is not review: it catches breakage, not bad design, missing tests, or a plausible misreading of the task. We are on the lonely side of the industry, deliberately, and it is genuinely open whether landing needs a gate.
- **The parallelism ceiling.** The 2–4 writer default is a heuristic we chose, not a consensus we borrowed, and we have not tested where coordination actually breaks. The risk is worse than "slower": measured multi-agent setups degrade 39–70% on sequential tasks — some work gets *worse* when split. queen should learn to say "this task does not parallelize," and that judgment is unbuilt.
- **Task-shape similarity** for the skills rule of three is undefined — embedding similarity, or queen's judgment over descriptors? Deferred with the feature.
- **Bearer credentials on a stateless transport.** Short-TTL, operator-only mints are today's mitigation for credentials that nothing binds to the specific launch that requested them; MCP's stateless transport gives the daemon no session to bind to. Whether that ceiling is acceptable, or whether the root needs its own session-bound handshake, is open.
- **The existential bet.** A vendor shipping cross-vendor teams, or an existing orchestrator growing a real control plane, would compress our gap. The counter is speed and depth on the safety stack and on the terminal — being the layer no single vendor can be. If that stops being true, Hive is a very nice personal tool, and that is the honest floor.
