Communication fabric · cross-cutting contract

Durable messages. Bounded context. Measured receipt.

Hive’s communication plane is a local, durable control system for agentic engineers. It identifies the real actor, authorizes the route, separates work from conversation, keeps durable truth lossless, and places only the relevant working set into provider context.

01 · Decision

Communication is a cross-cutting contract, not a fourth transition.

The terminal transition supplies reliable session transport. The Queen’s Hive supplies identities, authority edges, tasks, and channels. Split Horizon consumes their typed projections. This document defines the seam that all three share.

C0 · TERMINAL

Make delivery truthful

Introduce stable address binding, MessageEnvelope v2, immutable receipt events, token-and-byte-bounded wakes, one serialized delivery lane, and per-boundary Hive context attribution while the flat assignment policy still governs.

C1 · HIERARCHY

Make routing accountable

Authorize every sparse route through the current hierarchy revision, task ownership, delegation chain, or explicit channel. Add artifacts, delta rollups, recovery, and per-edge byte/token bounds without replaying history when an edge opens.

C2 · UI

Make state legible

Expose derived queue, receipt, channel, artifact, failure, and token-attribution projections in the native AppKit workspace. The UI never invents a stronger state than the daemon ledger contains.

Ownership boundary. Terminal Stack Transition owns bytes, sessions, provider adapters, safe injection, and raw provider-usage observations. The Queen’s Hive owns who may communicate, about which task, under which authority. This document owns the envelope, MCP operations, durable delivery ledger, context selection, Hive-added input attribution, and receipt semantics. Split Horizon owns presentation and engineer controls.

02 · Invariants

Identify. Authorize. Deliver. Prove.

Messages are untrusted content carried by a trusted control plane. Natural language may explain an action; it may never create authority, change lifecycle, or stand in for evidence.

01 / IDENTIFY The daemon names the actor

No caller supplies from. The authenticated action binding determines the sender Address, the exact process binding, and the resolved MessageScope.

02 / AUTHORIZE The route is a capability

A recipient name is not permission. The daemon resolves a target to a stable Address, checks instance/Assignment/capability authorization at C0 — owner edges, channels, and fences at C1 — and checks again before delivery and read.

03 / DELIVER One lane owns each recipient

Inbox pulls, provider-native delivery, session injection, human composer leases, and control traffic serialize through one recipient lane. Competing transports never race.

04 / PROVE Events, not optimistic labels

Queued, transport-written, provider-observed, recipient-acknowledged, and applied are different facts. A timeout, successful command, or later turn is never promoted into proof of action.

03 · Ground truth

Keep what is load-bearing. Replace what is ambiguous.

Hive already has the right bones: SQLite durability, provider-aware delivery, composer protection, explicit priority, bounded root envelopes, and measured provider boundaries. The redesign removes name-only identity, mutable receipt ambiguity, unbounded body injection, and policy hidden inside transport code.

04 · Architecture

One owner for each truth.

The communication plane is deliberately local. Hive is not inventing an internet agent-discovery protocol; a future A2A gateway may translate at the boundary without exposing internal grants or weakening local receipt semantics.

Daemon control plane

  • Identity registry: reserves agentId and generation before provider spawn and binds credentials to the launched process.
  • Policy engine: evaluates MessageScope, instance/Assignment, and capability bindings at C0 — hierarchy/task/channel/run fences from C1 — at admission and use.
  • Message ledger: stores immutable envelopes, decisions, attempts, receipts, and terminal failure reasons.
  • Scheduler: enforces per-recipient order, fair bounded queues, backpressure, expiry, and a separate control lane.
  • Context assembler: selects pinned truth, the active working set, and referenced history; records every Hive-delivered block before it crosses a provider boundary.

Execution and presentation

  • Provider adapters: expose only the boundaries Claude, Codex, or Grok can actually prove.
  • Usage observers: normalize provider-reported input, output, cached input, and reasoning where available; missing fields remain unknown and no price is inferred.
  • sessiond: owns Ghostty process transport and safe terminal input after the terminal transition.
  • Hierarchy services: own Task, InterfaceContract, ArtifactRef, DelegationGrant, SpecRevision, SpawnBrief, and CommunicationChannel authorization.
  • Workspace: renders projections and sends typed engineer commands; it never writes ledger state directly.
Control fields are not prompt text. Sender, recipient, MessageScope, priority, intent, evaluated policy revisions, expiry, and message-content/artifact permissions live outside the message body. Providers receive the body inside an explicit untrusted-content boundary.

The manifest journal is the durable truth

The normative copy of every WorkManifest revision is an append-only, checksummed manifest journal in Hive-owned storage outside every worktree, written with atomic compare-and-swap revisions. SQLite holds only a query projection that startup can rebuild from the journal. A manifest row is independent of the agent/worktree registry — no cascade ever deletes it — because the disaster window this record exists for is exactly when mutable registry rows have historically been lost.

Write cadence is event-driven

The daemon appends a manifest revision on spawn/Assignment, on each accepted status or progress checkpoint, on observed Git ref/commit and file-watcher changes, on promotion, and at final teardown capture. A bounded reconciliation sweep is evidence maintenance only. Watcher overflow or an inaccessible worktree sets completeness = unknown; unknown never means empty.

Identity is not authority. An operator message records engineer guidance; prose such as “approved,” “land,” or “stop” never performs G1, G2, lifecycle, or run control. Those remain typed daemon operations. Direct human bytes sent to a provider TUI are terminal input-arbiter transactions outside the message ledger; a Workspace/CLI composer message enters the ledger as operator. Stable identity before spawn

The daemon reserves agentId, generation, display name, node slot, and credential binding synchronously before launching the provider. Messages may target that exact reserved identity. If spawn fails, the reservation and its queued messages become terminal rejected; none may attach to a later reuse of the name. An ordinary envelope addressed to the logical root remains queued across queen provider succession. Generation-bound control does not transfer: its DeliveryAttempt targets one exact root binding and becomes terminal or explicitly recovered when that binding is replaced.

Causality without an exactly-once fiction

Idempotency keys are scoped to the sender plus the resolved message scope: an agent key scopes to agentId + generation + scope, and a logical-kind key (operator, root, system) scopes to the exact authenticated action binding. inReplyTo and causal parents express dependency; the recipient sequence expresses ledger admission order. Inbox and event consumption are at least once — consumers dedupe and acknowledge by messageId and envelope digest. A provider delivery attempt is never blindly repeated after transport-written or in-doubt: retry requires adapter-proven idempotency, otherwise the fabric reconciles from evidence or defers to the owner or engineer.

Message ≠ task ≠ context ≠ artifact. A message may negotiate or update a task. A thread groups related conversation. A task owns durable work and termination. An ArtifactRef names immutable or revisioned output. Completing a task is terminal; later refinement creates a new task linked to the same run or thread.
Message content is not a work artifact. Inline envelope content is capped at 16 KiB. Overflow uses a MessageContentRef plus a declared length, media type, and at most 1 KiB of inline preview so a 4 KiB wake remains triageable. Delivery never pastes referenced content into a terminal; recipients pull authorized ranges. If content must remain useful after the message is consumed, publish a hierarchy-owned ArtifactRef from C1 onward. Hive never auto-promotes message content into a work artifact.

06 · Routing policy

Default-deny, with narrow channels for real collaboration.

At C0 the flat Assignment binds each worker to the queen. From C1, the queen rules through revisioned owner edges: the queen owns direct and flat workers, and a lead owns its crew when the full hive is active. Routing follows the current owner by default, while explicit paired and cross-boundary work uses narrow channels rather than a permanent all-to-all chat mesh.

Self-hosting proof. A queen-brokered pair channel—two equal agents, one shared design task, explicit owner, bounded lifetime—is the canonical first worker-to-worker channel. The protocol must support the collaboration used to design it.

Revalidate on use

Admission records the policy that allowed a message. The daemon re-evaluates the current exact invoking binding and, for delivery, the exact recipient binding, then MessageScope authorization, capability epoch, and — at C1 — channel lifecycle, hierarchyRevision, and runEpoch before delivery, message-content/artifact read, or acknowledgement. Revocation can stop a queued message without rewriting the original decision. A paused run fences every work-bearing action — admission, delivery, reads, spawn, write, promote — while typed control, recovery, acknowledgement, and evidence traffic remain available. A historical action stays valid evidence if its epochs were valid when it occurred and no explicit later invalidation names it; a new invocation always evaluates current state. Critical revocation may explicitly invalidate pending candidates, verdicts, grants, messages, and attempts.

No implicit broadcast

Fan-out duplicates cost and creates ambiguous partial receipt. Owner and queen updates use typed rollups or explicit bounded recipient sets expanded by the daemon into independently addressed envelopes. Every recipient gets its own sequence and receipt trail.

Attribution boundary. Where an adapter can count the final rendered input, Hive compares that input with an otherwise identical rendering that omits Hive-owned blocks; the marginal difference is Hive-delivered input, and any tokenizer-boundary residual is protocol overhead. The ordered block manifest allocates that result to payload, protocol, and replay. Usage is not an agent tool. TokenAttributionProjection is a read-only Workspace/CLI daemon projection and is not loaded into provider tool schemas or model context. Observing whether Hive is efficient must not itself add recurring Hive tokens.
Tool context is context. Each recipient receives only the small authorized core schema needed for its current work. Optional Hive tool families are deferred or discovered on demand where the provider supports it; unsupported adapters use a deliberately bounded static set. Tool discovery changes visibility, never capability or authority.

Typed work remains typed

Assignment status, task transitions, reviews, promotion requests, G1/G2, budget changes, and pause/abort use hierarchy or run-control operations. A sentence such as “done,” “approved,” or “land this” has no lifecycle or authority effect.

Artifacts stay out of the queue (from C1)

hive_publish_artifact verifies the caller’s task/worktree scope, hashes the content or commit object, and returns the hierarchy-owned ArtifactRef. hive_read_artifact opens a bounded range after authorization. Messages carry summaries and references, not megabyte payloads or arbitrary filesystem paths.

Spawn is not send. At C0 hive_spawn launches with the current Assignment-scoped spawn brief; C1 accepts a typed Task/DelegationSpec request and records the hierarchy-owned SpawnBrief. Either way the daemon reserves identity first, launches the provider with that exact brief as process input, and measures the first boundary. Failure terminates the reservation and brief binding. Chat messages may refine an active Assignment at C0 or Task from C1; they never substitute for its initial contract.

08 · Wake and delivery

Wake cheaply. Pull deliberately. Interrupt truthfully.

A provider wake is a scarce context event. The daemon batches ordinary traffic, preserves urgent semantics, and records provider limitations instead of pretending every adapter has the same control surface.

Wake budget. A wake envelope is at most 4 KiB and 512 rendered tokens; the lower limit wins. It contains a batch cursor, counts, message IDs that fit, and bounded previews. Overflow remains queued for the next pull rather than expanding the wake. Full content is pulled only when needed. Critical explanation remains self-contained, is at most 2 KiB, and must also fit the rendered-token ceiling; the control operation itself is typed outside the body.

Priority is not intent

  • pause: advance the relevant authority epoch, capture the process tree, verify SIGSTOP of the same process, and resume only after re-attestation.
  • stop / cancel / restrict-writes: revoke authority, reap or replace the process according to teardown policy, and verify the resulting state.
  • invalid combinations: reject them at admission; do not infer intent from message prose.

One recipient lane

  • Recheck the human composer lease immediately before every write or provider call.
  • Serialize provider-native and sessiond paths; a message chooses one adapter per attempt.
  • Keep control traffic independent of normal queue depth.
  • Preserve provider-specific truth: Claude/Codex steer at tool boundaries; unsupported adapters say they deferred.

09 · Receipts and failure

Absence is unknown. Contradiction is evidence.

The message projection is derived from positive controls. A quiet terminal may be idle, modal, disconnected, dead, or simply unsupported by the observer. Hive preserves that uncertainty and escalates only from measured facts.

EventWhat it provesWhat it does not prove

Busy, modal, deaf, and dead differ

Turn state and process liveness are separate axes. The delivery scheduler uses provider events, process identity, session state, composer lease, and contradictory observations together. A single empty reader or stale screen is never enough to classify the recipient.

Alert once, with evidence

Missed urgent deadlines, terminal recipient loss, queue pressure, and adapter degradation append durable incidents. Acknowledgment deadlines start at the matching transport-written event, never enqueue. Initial policy is urgent 180 seconds and critical 10 seconds. A miss appends typed attention and invokes intent-specific escalation/recovery; it does not rewrite any receipt or permit blind retry. At C0 the queen receives instance and Assignment aggregates; C1 adds owner-subtree and run rollups. Repeated maintenance polls do not flood agent context with identical alerts.

Stranding is measured, not inferred

Startup after an unclosed daemon epoch reads the manifest journal first, rebuilds the projection, and marks every active manifest stranded from journal state alone — before any worktree is opened. Hive worktree registrations and the worktrees directory are enumerated only to detect existence and missing/orphan cases, never to reconstruct task context. A worktree with no manifest is a surfaced incident, never silently adopted or deleted. The one permitted on-disk safety measurement exists to prevent unsafe cleanup; it is not forensic reconstruction and never overwrites the journaled handoff.

Reconciliation and the Engineer prompt

Auto-clean requires both a known-complete manifest and an agreeing safety measurement taken with untracked-files=all semantics: zero unique commits relative to the recorded base, zero dirty files, zero untracked files, and no recovery artifacts or pending promotion. The cleanup itself is bound to that exact manifest revision, head, and watcher watermark under compare-and-swap after the measurement — any intervening change preserves and prompts, closing the measure-then-delete race. Any disagreement, unknown, or content preserves the worktree and raises a typed Engineer prompt rendering the goal/task/brief, owner and generation, exact commits with summaries, dirty/untracked list, last progress/decisions/evidence, and freshness/cause. Typed choices: resume/complete (a fresh agent binding and generation spawned with a brief citing the WorkManifest), preserve, or discard (explicit destructive authorization). Completion still follows ordinary promotion and landing, never an ad hoc cleanup. Before Split Horizon the prompt is a CLI/daemon surface. Commit subjects, dirty paths, and agent-written handoff fields are untrusted measured/claimed data: the prompt and the successor brief render them outside control fields, they can never override pinned engineer constraints, mint authority, select a disposition, or become instructions merely because they sit in a WorkManifest — the successor verifies them.

10 · Context and flow control

The durable truth is lossless. Active context is selective.

Multi-agent communication carries a measurable token and attention tax. Hive reduces repetition and irrelevant context, never the only copy of information the engineer or agent may need.

Three internal context classes

  • Pinned truth: exact active engineer instructions, requirements, acceptance criteria, authority, unresolved decisions, blockers, and uncertainty survive every compaction and handoff.
  • Active working set: current objective, recent relevant deltas, evidence, failures, and next action are revised in place instead of appended repeatedly.
  • Referenced history: old messages, completed decisions, transcripts, logs, files, diffs, and tool output remain durable and boundedly retrievable by ID, revision, digest, and range.
  • These are deterministic context classes, not user-selectable quality or usage modes.

Bounded delivery

  • Per-recipient message-count, byte, and rendered-token limits; C0 bounds instance and Assignment scopes, C1 adds channel and run bounds.
  • One ordinary wake per available provider boundary, containing a bounded batch.
  • Opening a channel or adding a participant does not inject prior channel history; the recipient receives only its brief plus novel deltas and exact references.
  • Fair scheduling across eligible scope senders after the isolated control lane.
  • Explicit backpressure with retry guidance; no silent drop and no unbounded SQLite queue.
  • Expiry for advisory traffic; durable task and control records use their own lifecycle.

Valuable information cannot hide behind a preview

  • New operator instructions, changed requirements or acceptance criteria, authority changes, direct work requests, decisions, review verdicts, blockers, and changed uncertainty are mandatory reads.
  • A mandatory item is delivered in full when bounded, or the recipient must pull and digest-check its exact content before acknowledging or continuing affected work.
  • The daemon classifies mandatory reads from typed sender, intent, task, review, and authority state; it does not spend another model call guessing importance from prose.
  • Receipts, unchanged status, duplicate revisions, and low-value lifecycle chatter stay outside provider context unless explicitly requested.

References carry the weight

  • Summaries cite message IDs and digests, MessageContentRefs, Assignment evidence, commits, diffs, tests, and decisions at C0; C1 adds Task revisions and ArtifactRefs.
  • Rollups contain deltas since the receiver’s acknowledged snapshot, not repeated full state.
  • Agents send changed decisions, evidence, blockers, and requests—not restated briefs or full conversation history.
  • Large content is range-readable and digest-checked.
  • Messages do not duplicate source files, full transcripts, or status already available through typed tools.
  • Content IDs, revisions, and digests suppress unchanged replay; tool results return requested fields or daemon-computed aggregates rather than full logs.
  • Uncertainty and blockers survive summarization; the rollup may not collapse “unknown” into “green.”
Context compaction is explicit. A structured checkpoint records its source message range, the resolved MessageScope revisions (Assignment in C0; task/tree/channel in C1), MessageContentRef/ArtifactRef IDs and digests as available, unresolved decisions, blockers, uncertainty, and omitted byte/token counts. Pinned truth is restored exactly. Provider-native compaction may reduce active context but its opaque output is never Hive’s only recovery record; the checkpoint supplements the durable ledger and never deletes the only copy of a pending instruction or silently changes the engineer-approved specification.
Handoff shape (SPEC §7). A compaction or replacement handoff carries three layers: pinned engineer constraints byte-identical to their approved source; daemon- and worktree-computed state as pointers and digests, never re-typed; and a short written layer of goal, done, failures with reasons, uncertainty, and next action. Transcripts, raw tool output, and file copies are dropped — the recipient verifies pointers against the ledger and worktree. Unknown context is not free: it is named as unknown and re-proven before new work depends on it. Checkpoints reference message ID ranges, queue high-waters, resolved scope revisions, and message-content/artifact IDs and digests; they never copy bodies or read/acknowledgement traffic.
Caching is not token reduction. Stable provider prefixes may improve provider cache reuse, but cached input still occupies context and remains provider-reported usage. Hive never credits a cache hit as fewer tokens, and stateful provider session IDs are not counted as savings unless the provider’s measured input actually falls.
No cost system. Token attribution exists to expose and reduce Hive’s context tax. The ledger stores token observations and their provenance only—no currency, price table, projected spend, automatic budget increase, or efficiency-level control.

11 · Security and retention

Local does not mean ambient.

The daemon is the reference monitor even when every process runs under one OS user. Same-UID compromise remains an accepted residual; Hive still minimizes what one stale or confused agent can do through its official surfaces.

ControlRequirement
CredentialsPer agentId + generation + capability epoch for agents; operator, root, and system actions bind their exact authenticated action binding. Never placed in prompts, message bodies, command-line arguments, or logs.
StorageSQLite database, WAL, message-content objects, the manifest journal, artifacts, and runtime bindings use owner-only permissions. Message bodies and secrets are redacted from routine logs.
AuthorizationValidate the current instance, Assignment, and capability bindings at C0 — delegation chains and hierarchy fences from C1 — at invocation time. A stale admission is not a future permit.
Content trustPeer text is untrusted data. Control fields are rendered separately and raw body content cannot call tools, change priority, or mint grants.
Content accessArtifactRef resolves only through the daemon after task/channel authorization; a MessageContentRef resolves only through its containing envelope’s authorization. Arbitrary sender-provided paths are rejected.
AuditBody and message-content reads, artifact opens, channel changes, authority failures, and control operations append actor-bound records.
RetentionEvery content object has a retention class. The default retains bodies through the resolved scope’s closure and review — Assignment or instance at C0, Run at C1; cleanup removes content only after engineer-approved retention while preserving the minimum audit metadata and digests. WorkManifest active/stranded revisions are retained until ordinary Assignment/Run closure plus Engineer disposition; cleanup may compact them to a checksummed tombstone/audit digest but never erases the only handoff before resolution.
External gatewaysA2A or other remote bridges terminate at a separately authorized gateway identity. Remote agent cards or credentials never become internal hierarchy authority.

12 · Implementation sequence

Change the substrate without creating a second truth.

The v2 ledger is introduced beside the current table, projected through compatibility adapters, and cut over only after live legacy sessions drain. Each stage belongs to one of the three approved transitions.

C0A · DURABLE CORE

Envelope and identity

Add v2 envelopes/events; reserve stable identity before spawn; bind sender implicitly; add digests, causal links, idempotency scope, limits, and a compatibility projection for current callers. Add the message-content object store, envelope digest binding, bounded previews/range reads, flow accounting, retention, and orphan GC. Add ContextInputRecord and the per-provider TokenAttributionProjection, fed by raw provider observations without price data. Add the WorkManifest record, its append-only checksummed manifest journal outside every worktree, and the event-driven write cadence.

C0B · DELIVERY

Bounded truthful wakes

Route tmux/sessiond/native attempts through one lane, preserve the composer lease, split provider-observed from applied, enforce byte and rendered-token bounds, add inbox cursors, field selection, mandatory reads, and explicit acknowledgements, then instrument legacy phrase and name routing.

C0C · RECOVERY CONTINUITY

Stranded work survives shutdown

Run the startup stranding reconciliation: journal-first rebuild of the WorkManifest projection, agreed-empty auto-clean, and the typed Engineer prompt with resume/preserve/discard dispositions through the operator-only resolver.

C1A · POLICY

Hierarchy-aware routes

Add current-revision edge checks, owner aliases, channels, delegation validation, run/task binding, emergency escalation, artifacts, rollups, budgets, and recovery routing. Bind WorkManifests to formal Task, SpawnBrief, and RecoveryCheckpoint records.

C1B · CUTOVER

Remove ambiguous paths

After no live legacy binding remains, reject caller-supplied from, name-only durable addresses, inbox-implies-applied, unbounded full-body paste, prose-derived control intent, and the old whole-orchestrator “Hive control” attribution.

C2 · WORKSPACE

Expose projections

Render queue pressure, stable participants, delivery timelines, channel expiry, artifacts, in-doubt actions, adapter degradation, provider totals, and Hive-delivered payload/protocol/replay input. Engineer controls call typed MCP/daemon operations; there are no cost controls or efficiency modes.

No flag-day message migration. A message is wholly v1 or v2 and carries its protocol version. Compatibility code may project v2 facts into old fields; it may not fabricate v2 receipts from old timestamps. The token UI is a hard semantic cutover: existing provider collectors may supply raw observations, but the whole-orchestrator “Hive control” projection and old Usage view are removed rather than shown beside TokenAttributionProjection. Restart recovery drains critical v2 control before ordinary traffic.

13 · Verification

The test suite attacks the boundaries.

Unit tests prove schemas and policy. Integration tests use real provider/session boundaries where the claim depends on them. Chaos tests interrupt the daemon between every durable step.

Identity and authority

  • Reserve identity, queue a message, fail spawn, reuse the display name: the successor receives nothing.
  • Admit under a valid edge, revoke before delivery: delivery is rejected with both policy revisions preserved.
  • Expire a pair channel with pending traffic: no new delivery; owner and sender see one terminal reason.
  • Attempt spoofed sender, cross-run read, stale generation acknowledgement, and artifact path escape: all fail closed.
  • Address queen without creating an agents row; across root succession, ordinary queued traffic survives and is reauthorized after attestation while stale generation-bound control is rejected.
  • Deliver a flat C0 envelope with no hierarchy record present: the full lifecycle works and nothing fabricates a Run or Task.
  • Reconstruct a worker or queen replacement from the recorded brief/checkpoint and measured replies; verify the exact context provenance and ensure no old generation or display-name successor can claim it.

Delivery and recovery

  • Busy, idle, modal, detached, dead, and composer-active sessions produce distinct measured projections.
  • Crash after admission, after transport write, and before receipt append: retry is idempotent or becomes in-doubt, never optimistic success — and no attempt blindly repeats after transport-written or in-doubt.
  • Claude, Codex, and Grok expose truthful steer/cancel degradation; no adapter test double substitutes for live acceptance.
  • Pause preserves and verifies the same process; stop/restrict-writes follows teardown and capability fencing. A paused run blocks work-bearing admission and delivery while control, recovery, acknowledgement, and evidence still flow; historical receipts with then-valid epochs remain evidence unless explicitly invalidated.
  • Normal batches respect recipient order, fair channel service, queue caps, and the 4 KiB / 512-rendered-token wake budget.

Context and evidence

  • A 1 MiB message becomes one MessageContentRef with a bounded preview in C0; from C1, a durable reusable work product becomes an ArtifactRef instead. Neither path terminal-pastes the body.
  • Inbox read does not become applied; explicit acknowledgement without matching digest is rejected.
  • Rollup compaction preserves exact pinned truth, blockers, uncertainty, source ranges, revisions, and omitted byte/token counts.
  • Every mandatory item is fully delivered or exact-content-read before affected work continues; preview-only delivery cannot satisfy it.
  • Duplicate idempotency keys return the original message only within the same sender binding and resolved message scope.

Token efficiency without quality loss

  • For each supported provider, reconcile provider-reported input/output with ContextInputRecords and prove Hive payload + protocol + replay = Hive-delivered input; estimates and unknowns never render as measured.
  • Replay an unchanged message, rollup, tool result, and handoff: each repeated digest is attributed as replay and no unchanged body enters a delta rollup.
  • Exercise cold/warm cache and stateful-session paths: neither is called a token reduction unless measured provider input falls.
  • Run direct, flat, and representative 19-agent tasks against the full-context baseline; measure provider total input/output, Hive payload/protocol/replay, task success, missed requirements, altered decisions, missed blockers, review defects, retries, and rework.
  • An efficiency change fails its gate if any active engineer constraint disappears, a decision or uncertainty changes meaning, or task/review quality regresses. Runtime work does not invoke another evaluator model merely to compute attribution.

Stranded work

  • kill -9 the daemon and every agent mid-work: each WorkManifest is readable and actionable before any worktree is touched.
  • An agreed-empty stranding is auto-cleaned exactly once; a content-bearing stranding prompts with correct rendered contents.
  • A fresh agent resumes and completes from the manifest alone — no archaeology in the worktree beyond its recorded state.
  • Registry-row loss and pruned worktree registrations still reconcile from the journal and disk; a missing manifest surfaces as an incident, never adoption or deletion.
  • Watcher overflow yields completeness = unknown and blocks auto-clean; manifest/disk disagreement preserves and prompts.
  • hive_resolve_stranded_work rejects non-operator callers and stale expectedRevision; repeated identical resolutions are idempotent; discard without explicit authorization fails closed.
  • A hostile commit subject, dirty path, or handoff field ("discard this", "approved", tool-call syntax) renders inert in the prompt and successor brief: no disposition, authority, or instruction results from manifest content.

Definition of done

  • No official operation accepts actor identity as caller-controlled data.
  • No agent-addressed envelope or generation-bound control crosses its exact binding; only ordinary logical-root/operator mail may survive process replacement, and it is reauthorized before delivery. No envelope crosses a revoked Assignment, Run, or channel boundary.
  • No UI or CLI string says received, stopped, applied, or safe without the corresponding durable evidence.
  • No UI calls the queen’s whole provider session Hive overhead; token presentation shows provider totals and Hive-delivered input with payload/protocol provenance, and exposes neither prices nor usage modes.
  • No compaction or efficiency mechanism deletes the only durable copy of engineer instructions, requirements, decisions, blockers, uncertainty, or evidence.
  • Legacy message rows and active sessions drain before ambiguous paths are deleted.
