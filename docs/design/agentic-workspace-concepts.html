Workspace redesign · three concepts

From tiled panes to operational truth.

Hive’s UI should behave like an engineering control plane, not a terminal window manager. The hierarchy is the primary object; terminals are durable resources selected for intervention.

Binding design move

Scan the system.
Attach only to act.

Structured events carry health, lineage, gates, queues, review, and lifecycle. One selected terminal is live; everything else remains observable through cheap snapshots and event state.

Supports queen + 3 leads + 15 workers without 19 live renderers.
01 / STRUCTUREThe tree outranks the pane

Parent, role, owner, generation, task, and gate remain visible as durable system identity.

02 / SIGNALStatus is not screen scraping

Lifecycle, attention, queue age, CI, review, and branch freshness come from typed events.

03 / CONTROLThree states stay separate

Keyboard focus, viewed scope, and write authority each have an explicit visible indicator.

04 / SAFETYDetach is ordinary

Closing or switching a view detaches. Termination is a distinct, authorized, previewed action.

The design space

One data model, three levels of disclosure.

The concepts differ in how much context stays simultaneously visible. They deliberately avoid floating windows, freeform tiling, and terminal-first navigation.

01 · Single layerDense

Control Ledger

A flat, sortable operations table. The full run stays in one scan plane; the selected row opens a fixed inspector.

Best for → triage, audit, keyboard-heavy operators
02 · Dual layerRecommended

Split Horizon

A persistent hierarchy rail and run pulse surround one full working surface for the selected agent.

Best for → daily engineering, monitoring + intervention
03 · Three layersScale

Observatory

Run, crew, and agent layers remain visible together. Progressive drill-down exposes ownership without losing rollups.

Best for → many teams, program-level supervision

Layer 1 · Run

Shows phases, G1/G2, and aggregate health without terminal content.

Layer 2 · Crew

Shows lead ownership, workers, attention rollup, review flow, and branch age.

Layer 3 · Agent

Shows a coherent snapshot, identity, task evidence, and explicit attach action.

Scaling rule

Add crews horizontally; do not add terminal tiles. Nonselected sessions stay detached.

Recommendation

Build Split Horizon first.

It gives an agentic engineer a stable global picture and a serious local work surface without turning hierarchy into a diagram they must constantly navigate. Add the Control Ledger as an alternate run view later; reserve Observatory for multi-run scale.

01
It matches the control loop

Scan attention, select an agent, inspect typed evidence, attach only when intervention is warranted.

02
It keeps safety state in view

Focus, viewed scope, authority, and exact generation remain pinned above the terminal.

03
It scales through detachment

One live renderer plus structured fleet state supports the 19+ session requirement cleanly.

04
It has a clean implementation boundary

Hierarchy rail consumes control-plane events; workbench consumes one SessionHost attachment and durable artifacts.
