PRODUCTION DESIGN · TERMINAL STACK · 2026-07

Terminal stack transition

One writer. Any viewer.

Hive replaces tmux vocabulary with a durable, generation-safe SessionHost. A per-tenant session daemon owns terminal truth; SwiftTerm and Electron become disposable clients.

Decision · adopt the research pivot

Do not ship the unchanged Zellij spike.

Zellij 0.44.3 is not a safe common Windows substrate. Hive owns the PTY protocol and single-writer queue; qualified Zellij is a Unix durability layer only, while Windows uses direct ConPTY.

01 · Decision frame

Move persistence below the UI, and policy above the PTY.

The migration is a host replacement first and a workspace replacement later. Those are separate failure domains, flags, and rollback decisions.

PARITY65

FIX15

Chosen architecture

A dedicated, per-HIVE_HOME hive-sessiond process exposes SessionHost over authenticated local IPC. It owns PTY reads, the headless screen model, output sequencing, input arbitration, process containment, and the only write handle.

On macOS and Linux, a release-qualified bundled Zellij may own each vendor PTY while hive-sessiond owns the sole attach client. On Windows, hive-sessiond owns node-pty 1.1.0/ConPTY directly and contains each generation in a Job Object.

Why the unchanged spike is no-go

Zellij 0.44.3 has a native-Windows reversed multiline-paste report, no cache-directory override that proves same-user isolation on Windows, attach/lifecycle defects, and no official Windows ARM64 artifact. Sending automation through a plugin while humans use an attach PTY would also create two writers.

Binding conclusion. Zellij is never the policy boundary, never an input API, and never required on Windows.

It makes one production choice the research left open: the screen/replay implementation is a daemon-owned headless xterm state machine plus a bounded output journal, never Zellij dump-screen as control truth.

03 · Production architecture

Viewers reconnect to truth; they do not own it.

The control daemon authorizes lifecycle and hierarchy. The session daemon owns terminal mechanics. The substrate owns only the platform process primitive.

Process boundaries

  • One hive-sessiond per canonical tenant identity, supervised independently from the Workspace and control daemon.
  • Private mode-0700 Unix socket or Windows named pipe ACL; protocol-version handshake and short-lived capability token bind tenant, actor, viewer, and allowed operations.
  • The control daemon is the lifecycle/hierarchy authority and stores no raw PTY handle. It reconnects after its own restart.
  • On Unix, sessiond owns exactly one node-pty attach process to each qualified Zellij session. No client attaches directly.
  • On Windows, sessiond owns ConPTY and a generation-specific Job Object containing the vendor and auxiliary processes.

Terminal state and flow control

  • A headless xterm parser holds the authoritative current screen, dimensions, alternate-buffer state, cursor, and monotonically increasing output revision.
  • A bounded per-session output journal supports snapshot-plus-cursor reattach. Slow viewers are disconnected with a resumable cursor; PTY reads use high/low-water backpressure.
  • No prompt or terminal content enters general logs or hierarchy events. Journals are tenant-private, size-bounded, and generation-owned.
  • capture() returns a coherent ScreenSnapshot, not a transcript: text, rows, columns, alternate-screen flag, revision, and generation.
  • Resize is coalesced: settle layout → fit terminal → one rows/columns update. Initial dimensions can never be 0×0.

The production seam

The low-level seam stays small, but production returns typed evidence and reserves hierarchy queries/events now. TmuxSessionHost and DaemonSessionHost implement this exact interface.

  readonly snapshot: ScreenSnapshot;
  readonly output: AsyncIterable<OutputFrame>;
  claimHuman(): Promise<HumanLease>;
  writeHuman(lease: HumanLease, frame: HumanInputFrame): Promise<InputAck>;
  releaseHuman(lease: HumanLease, reason: "submitted" | "cancelled" | "explicit-handoff"): Promise<void>;

Generation-safe locator

SessionLocatorV1 is an opaque, canonical, MACed wire value. The schema package alone decodes it. Mutable names and paths are presentation metadata.

{
  v: 1,
  tenantId, hostEndpointId,
  hierarchyRunId,
  role,
  agentId,
  parent: { agentId, generation } | null,
  generation,
  taskId?: string,
  gateId?: string
}

Equality covers every field. A locator for an old generation returns generation_mismatch; it never follows a reused name. Authentication is separate from identity, so persisted locators contain no bearer secret.

Large prompts are streamed, not launched

SessionSpec contains structured argv/env and a generation-bound initial-input stream, not prompt text in argv. Sessiond spools bounded bytes to an unlinked/mode-0600 generation resource, starts the vendor TUI, then submits that input through the same acknowledged arbiter used for later messages.

The spool is deleted after verified handoff; an in-doubt crash never blind-retries. The gate is a hash-checked payload of at least 1 MiB with multiline Unicode/control content and no bytes in argv, process listings, general logs, session names, or crash reports. This removes both tmux's ~16 KiB command ceiling and the remaining ARG_MAX ceiling.

04 · Input authority

A latch, not a typing timer.

All human and automated bytes enter one per-generation queue. The hierarchy router may authorize a message; only the session arbiter may write it.

AUTOMATION_READY

FIFO automation may start. A human claim wins before the next transaction.

AUTOMATION_WRITING

One bounded paste/cancel/submit transaction is atomic; no interleaving.

HUMAN_OWNED

Latched before the first byte. Every automation priority queues. No timeout.

HUMAN_ORPHANED

Viewer vanished with possible draft. Fail closed until explicit, visible handoff.

Transitions

  • claimHuman fsyncs generation + viewer ownership before returning a lease; human bytes without that lease are rejected.
  • Typing, paste, IME, accessibility input, and programmatic terminal input all claim synchronously before their first PTY frame and then forward unchanged.
  • Submit, cancel, or an explicit takeover/handoff releases. Focus change, silence, elapsed time, output, renderer exit, and attach loss never release.
  • A viewer crash moves to HUMAN_ORPHANED. Reconnect shows “draft protected”; the human may continue or explicitly abandon. No automation priority can override it.
  • Termination closes the queue, records every unstarted/in-doubt transaction, then verifies process state.

Acknowledged automation

  • Each hierarchy-authorized envelope carries message ID, sender/recipient generation, ordering key, intent, correlation ID, and content digest.
  • Daemon receipts separate queued, writing, transport_accepted, tui_acknowledged, cancelled, and in_doubt.
  • Paste and submit are distinct ordered frames. The daemon waits on parser/PTY acknowledgement and vendor lifecycle evidence, never a fixed 500 ms sleep alone.
  • Existing hooks/artifact activity remain the proof of TUI receipt. Transport acceptance maps to “injected,” not “applied.”
  • Urgent Escape → clear → paste → submit remains one required transaction shape. A typed intent cannot be structurally dropped.

05 · Attach and recovery

Snapshot, cursor, then ordered frames.

Reconnection is explicit about what survived. Hive never calls a relaunched provider process a reattached terminal.

Client attach protocol

  1. Client connects to the tenant-private endpoint and negotiates protocol version, actor authority, viewer ID, and flow-control limits.
  2. ATTACH(locator, rows, cols, afterCursor) exact-resolves tenant + agent + generation. Not-yet-created, absent, stale generation, unauthorized, and unknown are distinct results.
  3. Server returns one atomic snapshot with output cursor, dimensions, buffer mode, lifecycle, attention, and input-ownership state.
  4. Ordered output frames begin strictly after the snapshot cursor. Gaps force a new snapshot; clients never guess.
  5. Input frames carry viewer sequence + digest and receive a highest-committed sequence ack. Reconnect can retry transport without duplicating committed input.
  6. DETACH or socket loss removes only the viewer. TERMINATE is a separate control-daemon-authorized operation.

SwiftTerm remains first client

TerminalPaneView.swift continues to host the real terminal. Instead of launching tmux attach-session, it launches an absolute bundled hive session attach <opaque-locator> bridge in its local PTY. The bridge translates PTY input/output, resize, detach, and human lease frames to sessiond.

This preserves the current geometry, mouse, focus, visible-screen smoke, and child-exit behavior while removing direct tmux access. Electron later connects from its sandboxed main process and forwards a narrow framed stream to xterm.js; renderer code never receives filesystem, node-pty, or bearer-token access.

10 · Conformance and release gates

One black-box suite, two hosts, no skips.

Before any host default changes, the current tmux adapter and new daemon host must demonstrate the same externally observable 65-capability contract.

Host matrix

  • Reference: TmuxSessionHost, wrapping src/adapters/tmux.ts and current process/lifecycle helpers.
  • Candidate Unix: DaemonSessionHost → qualified Zellij backend on macOS/Linux.
  • Candidate Windows: DaemonSessionHost → direct node-pty/ConPTY backend.
  • The identical parity cases compile against the interface, use real postcondition readback, and forbid backend-specific expected failures or skips.
  • Defect-closure, hierarchy, load, packaging, and real-vendor suites are additional candidate gates; they do not dilute parity.

Required lanes

  • Hermetic contract tests with deterministic fake vendor and fault injection.
  • Real tmux and exact bundled Zellij/ConPTY lifecycle integration tests.
  • Claude, Codex, and Grok TUI fixtures for keyboard, mouse, alt-screen, resize, capture, ack, detach, and recovery.
  • Crash matrix: renderer, attach bridge, UI/main, control daemon, sessiond, substrate, machine-restart reconciliation.
  • 19+ session load, sustained output, slow-viewer backpressure, queued automation, runaway descendant, and verified teardown.
  • Signed/package smoke on every OS/arch with user binaries absent from PATH and hostile configs present.
Default gate. No default flip until: identical parity suite is green on both hosts; all 15 defect closures are green on the candidate; the target platform substrate is release-qualified; SwiftTerm has completed a canary cycle; rollback has been exercised with live generations draining on their original hosts.

11 · Implementation plan

Eight phases, four independent flags.

Every phase has a measurable entry, exit, and rollback. Flag changes affect new admissions or client selection only; they never transplant a live generation.

P0

Freeze contracts and evidence

Introduce neutral schemas and the conformance harness without changing runtime behavior.

  • Add SessionHost, locator/spec/inspection/event/input types, golden serialization, and capability IDs A01–E12.
  • Build TmuxSessionHost from the current adapter; record reference traces.
  • Specify DB/feed dual-read: sessionLocator preferred, tmuxSession legacy.
EntryInventory/research landed and current tests green.
Exit65 reference cases pass tmux; schema compatibility round-trips.
Rollback: remove unused seam/schema before any writer exists. Risk: abstraction hides postconditions. Control: cases assert state, not calls.
P1

Build sessiond and attach protocol

Create the per-tenant process, authenticated IPC, sole-writer arbiter, output journal/headless screen, and direct test PTY.

  • Implement snapshot + cursor, sequence acks, flow control, human latch WAL, initial-input stream, event cursor, and typed errors.
  • Add supervisor/handshake and private runtime endpoint.
  • No production agent is admitted yet.
EntryP0 contract frozen.
ExitFault/crash tests prove viewers detach, no second writer, no timed release, bounded memory.
Rollback: terminal_session_host_v2=off; stop unused sessiond. Risk: IPC/WAL duplicates input. Control: sequence/digest fault injection.
P2

Implement and qualify substrates

Add the Unix Zellij backend and Windows ConPTY backend behind sessiond; port process/resource/teardown inspection.

  • Pin Zellij 0.44.3 candidate, xterm.js 6.0.0, node-pty 1.1.0 and checksums/provenance.
  • Run the identical 65-case suite against tmux and new host plus all defect/platform gates.
  • Unify initial and recovery readiness.
EntryP1 protocol stable.
ExitAll parity + defect cases green for each target; Zellij release record signed off.
Rollback: keep flags off; retain last qualified pin or direct test backend. Risk: Zellij fidelity/daemon crash or ConPTY quirks. Control: real-vendor and crash matrix.
P3

Retain SwiftTerm, replace its attach path

Ship the bundled session attach bridge and dual-protocol pane model before changing which host creates sessions.

  • Update TerminalPaneView.swift to observe input → claim lease → forward unchanged through bridge.
  • Preserve real geometry, mouse, focus, scroll, visible text, and child-exit semantics.
  • Change ordinary app quit to detach; add explicit Stop Hive.
EntryP2 candidate green.
ExitSwiftTerm can view/control tmux and daemon generations in one workspace; clean quit survival passes.
Rollback: terminal_swiftterm_attach_v2=off for tmux generations; daemon generations keep the bridge and drain. Risk: missed AppKit input path. Control: typing/paste/IME/accessibility tests.
P4

Shadow, opt in, canary

Run read-only shadow inspection first, then admit explicit new-host canaries while tmux stays default.

  • terminal_session_host_v2=shadow compares list/inspect/capture/resource results without shadow writes.
  • on is tenant/cohort opt-in for new generations only.
  • Exercise real rollback: new admissions return to tmux while existing daemon sessions remain serviced.
EntryP3 mixed-host SwiftTerm ready.
ExitCanary SLOs and 72-hour durability/load pass; rollback drill produces no lost/misrouted input.
Rollback: change admissions to tmux; never migrate live PTYs. Risk: dual-host recovery kills wrong generation. Control: locator exactness and per-host offline cleanup.
P5

Flip host default and drain tmux

Make sessiond the default per qualified platform while SwiftTerm remains the default workspace.

  • New sessions use daemon host; existing tmux locators continue attach/inspect/terminate.
  • Monitor receipt latency, reconnect, input in-doubt, host loss, survivor, resource, and stale-locator rates.
  • Stop creating legacy tmuxSession; retain compatibility reader and offline cleanup.
EntryP4 canary/rollback accepted.
ExitTwo stable releases with daemon default and zero unmanaged tmux generations.
Rollback: set terminal_session_host_v2=off for new admissions; daemon generations drain. Risk: partial tenant rollout. Control: host kind persisted in every locator.
P6

Gate Electron workspace separately

Build tree layout, xterm.js 6.0.0 rendering, sandboxed IPC, virtualization, and application navigation mode without coupling it to host rollout.

  • Main process holds daemon connection; renderer receives narrow output/input/resize events only.
  • Implement group tree, three-way focus/view/authority, attention rollups, and 19+ off-screen virtualization.
  • Run side-by-side client conformance against the same daemon generations.
EntryP3 attach protocol stable; may proceed after it independently of P4/P5.
ExitElectron client gates green and explicit cohort approval obtained.
Rollback: terminal_electron_workspace=off; reopen SwiftTerm against unchanged live generations. Risk: renderer fidelity/security. Control: vendor corpus, context isolation, no Node in renderer.
P7

Retire tmux production code

Remove the external prerequisite only after the drain window, while retaining a bounded legacy cleanup/import path.

  • Delete tmux admission and Workspace direct attach; stop packaging/documenting tmux.
  • Keep a standalone legacy detector/cleanup reader for one major release, default-home only.
  • Remove tmuxSession writes after migration telemetry proves no old readers; migrate schema last.
EntryP5 two-release drain, zero live tmux, rollback package tested.
ExitInstall/run tests pass with tmux absent; code search has only intentional legacy cleanup references.
Rollback: ship the supported prior release/adapter package; no live session transplant is needed because entry requires zero tmux sessions. Risk: stranded old default-home session. Control: one-major legacy cleanup window.

Risk is retired before the next irreversible step.

Every high-risk item has a measured control and an owner-facing stop condition.
