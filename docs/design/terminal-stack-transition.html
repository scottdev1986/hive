01 · Architecture decision

A macOS-only product earns a much simpler, better terminal.

Dropping Windows and Linux removes Zellij, ConPTY, Electron, xterm.js, native-module matrices, and asymmetric behavior from the product plan. It does not remove the need for session durability or strict input ownership.

PLATFORM1

macOS product target, with universal arm64 and x86_64 release artifacts.

PLANES3

Terminal bytes, agent control/state, and native presentation stay independent.

SCALE19+

Concurrent ordinary generations as a forward capacity and load target, before transient and recovery sessions.

GATES8

Ordered phases with measured exits and admission-only rollback.

Chosen stack

UI: the existing native AppKit workspace, evolved into Split Horizon. Renderer: upstream MIT libghostty, pinned to an exact commit and packaged as a universal XCFramework. Session host: a small bundled Zig service using macOS PTYs and headless libghostty-vt, with one broker and one isolated host process per generation. Control: the existing Bun daemon remains lifecycle, policy, hierarchy, and event authority.

What “Ghostty-based” means

Hive adopts Ghostty’s terminal parser, input encoder, grid semantics, and Metal surface as an internal engine. The product remains Hive: its own workspace, terminal profile, settings, keybindings, status model, hierarchy, and visual language. Hive does not fork the Ghostty application or copy its window model. A bounded compatibility bridge adds embedder-owned I/O and complete checkpointing until upstream exposes them.

Why not cmux

cmux proves the experience is feasible: native AppKit, Ghostty rendering, notification rings, command APIs, session restore, and agent-oriented affordances. It is an application, not a reusable terminal SDK; its terminal view is tightly integrated and its repository is GPL-3.0-or-later. Hive is MIT.

License boundary. Treat cmux as behavioral prior art only. Import no cmux source. Re-derive requirements from public behavior and implement against upstream Ghostty headers.

Why not stock Ghostty alone

Stock libghostty can create an AppKit surface and spawn a command on its own PTY. Hive needs the PTY to live outside the app, with output supplied by hive-sessiond and encoded input returned to that daemon. That manual-I/O seam is not in the current public header.

Critical gate. Renderer adoption can start with stock Ghostty launching a temporary hive session attach command. Direct replay and crash-safe ownership cannot ship until the small manual-I/O bridge passes its gate.

05 · Ghostty integration

Adopt the engine behind a narrow, replaceable boundary.

Ghostty’s terminal behavior is mature; its embedding C API is explicitly still in flux. The plan pins source, fences the API, and proves every unsupported assumption before product migration.

Use from upstream

  • libghostty AppKit surface and Metal renderer.
  • Keyboard, text, preedit/IME, mouse, paste, clipboard, selection, and screen-read APIs.
  • Unicode/grapheme and cell-width behavior, ligatures, true color, hyperlink, Kitty keyboard/graphics, synchronized output, and modern terminal modes.
  • libghostty-vt in sessiond for headless parsing, resize/reflow, screen state, and checkpoint formatting.
  • MIT license and upstream tests, fuzzers, and terminal corpus where reusable.

Hive bridge: allowed surface only

  • Create a surface in manual_io mode with no child PTY.
  • Feed ordered PTY output bytes into a surface.
  • Receive Ghostty-encoded input bytes through an embedder callback.
  • Inject plain text/paste through the same terminal encoder.
  • Read a render-grid/checkpoint diagnostic only where upstream lacks a stable equivalent.
Patch budget. One Hive public header, at most six upstream implementation files and 1,500 non-test lines, one Swift wrapper, and dedicated ABI tests. Any request for window management, agent features, process ownership, settings, or hierarchy in the fork is rejected.

Version and provenance policy

  • Production baseline: upstream commit 73534c4680a809398b396c94ac7f12fcccb7963d. T1 qualifies this exact source. Changing it requires a reviewed lock-file update, bridge rebase, ABI diff, and complete terminal corpus.
  • Track a Hive fork only while the manual-I/O change is not upstream; rebase intentionally, never float on main.
  • Produce GhosttyKit.xcframework for universal macOS in CI, verify exported symbols and header hash, cache by source and Zig toolchain hash.
  • Include Ghostty and transitive notices, source provenance, SBOM, checksums, code signing, notarization, and reproducible build metadata.

Hive terminal identity

Hive owns the complete terminal profile: font, size, palette, cursor, ligatures, opacity, keyboard commands, navigation, security, process launch, shell integration, clipboard policy, working directory, and renderer limits. Users configure these in Hive Settings using Hive terminology and defaults.

Binding product decision. Hive does not read or import Ghostty configuration files. Ghostty is not exposed as a parallel settings system, compatibility promise, or product identity. A future theme format must be a Hive format designed and versioned on its own terms.

06 · Native renderer experience

The TUI stays authentic while the renderer becomes replaceable.

T0–T7 replaces SwiftTerm inside the current AppKit pane topology; Hive owns the surface contract, not a new navigation model. The hierarchy rail, selected workbench, inspector, and global navigation are Split Horizon deliverables after The Queen’s Hive.

Attached surface

  • A surface attaches to exactly one live session by exact locator and generation.
  • The native title strip shows agent, role, task, provider/model, branch/worktree, status provenance, input owner, and connection health.
  • The Ghostty surface fills its pane and gets first responder only after an explicit click or keyboard navigation.
  • Unread output and attention are surfaced without stealing focus; nothing retargets an attached surface implicitly.
  • Detached agents remain fully alive in sessiond. Renderers are disposable and admitted against measured caps; surface count and retention policy belong to the embedding UI.

Native terminal affordances

  • Full keyboard, Option behavior, dead keys, IME/preedit, bracketed paste, drag selection, mouse reporting, and alternate screen.
  • Native menu commands for copy, paste, select all, search, clear scrollback, reset renderer, detach, and terminate.
  • Hyperlink hover/open with an explicit trust gesture; file paths can offer Quick Look where policy allows.
  • VoiceOver-accessible visible rows, selection, cursor, title, status, and input ownership.
  • Appearance settings live in Hive Settings and update surfaces without changing session state.

Attach and retarget sequence

  1. Resolve the target agent to an exact SessionLocator.
  2. Authenticate the local client and request attach with geometry and last acknowledged output sequence.
  3. Apply the newest valid terminal checkpoint, then every subsequent frame in order.
  4. Create or reveal the Ghostty surface; acknowledge the applied sequence.
  5. Only after explicit focus may the client acquire a human input claim.
  6. On retarget, release focus and renderer resources; do not alter the process or task. Output never retargets: a late frame can never reach a surface bound to a different locator or generation.

Disconnect semantics

  • Pane/window close: detach renderer.
  • Workspace quit: detach all renderers; Hive sessions remain.
  • Renderer fault: recreate the surface from a checkpoint.
  • Control-daemon restart: reconcile sessiond generations and provider state.
  • Sessiond restart: declare affected PTYs lost; never claim a reattach that macOS cannot provide.
  • Stop agent / Stop Hive: explicit confirmation, authorized termination, positive process readback.
Honest durability boundary. Moving the PTY out of the Workspace guarantees UI-crash survival. Each detached host owns exactly one macOS PTY, so a broker crash only disconnects viewers while the hosts continue and are adopted after restart. A host crash still destroys that one PTY; recovery creates a new generation from provider recovery data and labels the old generation lost.

Fusion and precedence

  1. OS/sessiond truth wins for process and session state.
  2. Provider hooks or app-server events win for turn and approval state.
  3. Task/review/gate records win for workflow authority.
  4. Agent reports fill descriptive phase, summary, progress, blocker, and evidence.
  5. Heuristics may produce a labeled low-confidence hint, never an authoritative value.
  6. Expired sources become stale or unknown; they are not held indefinitely.

Event delivery and the rollup seam

  • Replace Workspace feed polling with a resumable event subscription plus periodic snapshot reconciliation.
  • Sequence every event per instance and include entity revision for idempotent Swift reducers.
  • The subscription and reducers ship against today's flat queen→workers policy; no reducer requires a lead or reviewer to exist.
  • The Queen's Hive later adds parent/edge/rollup reducers on the same envelope — scoped summaries instead of terminal transcripts.
  • Engineer sees an honest aggregate with drill-down to the source evidence.

09 · Session protocol and durability

Reconnect is a protocol, not a hopeful redraw.

The protocol carries exact identity, ordered bytes, checkpoints, input transactions, geometry, and diagnostics. It is local-only and versioned from its first test.

Session locator

{
  "instanceId": "hiveInstanceSuffix(resolveHiveHome())",
  "subject": { "kind": "agent", "agentId": "agt_..." },
  "generation": 4,
  "hostKind": "sessiond",
  "sessionId": "ses_...",
  "engineBuildId": null

The display name, role, provider, task, branch, and worktree are metadata. None may be used to recover a missing locator.

Local transport

  • One broker Unix-domain socket plus generation-specific host endpoints inside the canonical private Hive runtime directory, mode 0700/0600.
  • Peer UID verification plus a short-lived operation capability issued by the control daemon.
  • Fixed binary header: magic, protocol major/minor, frame type, flags, payload length, and sequence.
  • JSON metadata for low-rate control frames; raw bytes for terminal output/input. Generated schemas keep Zig, Swift, and TypeScript aligned.
  • Frame and queue limits are negotiated but server-capped; malformed clients are detached without affecting the PTY.

Attach/replay contract

  1. HELLO / version / client role / capability.
  2. ATTACH with locator, rows, columns, pixel size, and afterSeq.
  3. SNAPSHOT at checkpoint sequence, containing complete same-engine restorable VT state.
  4. OUTPUT frames with strictly increasing sequence and monotonic timestamp.
  5. APPLIED high-water acknowledgement from the renderer.
  6. RESIZE, input-claim, input, inspect, detach, and explicit terminate transactions. User copy is viewer-local; paste follows the input transaction.

Checkpoint truth

Sessiond parses every output byte through libghostty-vt and maintains a bounded journal. A checkpoint is valid only if it can restore both primary and alternate screen, cursor, modes, palette, hyperlinks, keyboard mode, scrolling region, tab stops, and Kitty graphics references needed by the renderer.

No false shortcut. The current formatter API appears capable of serializing active terminal state, but it is not the complete checkpoint contract. Implement or upstream the missing state export and pass TG2 before sessiond admission. The bounded journal connects a valid checkpoint to live output; it is not permission to ship a “mostly restored” TUI.

Backpressure and resources

  • The PTY reader never blocks on a slow viewer; output first enters bounded session storage.
  • A lagging viewer is told to rebase from a newer checkpoint, then detached if it still cannot keep up.
  • Set separate caps for live queue, raw journal, scrollback cells, image bytes, checkpoint count, and disk spill.
  • Measure 19 normal agents plus transient reviewer/recovery sessions under worst observed output.
  • Expose queue lag, renderer lag, checkpoint age, RSS, disk use, and dropped diagnostic hints.

Process containment and recovery

  • Spawn structured argv, cwd, environment allowlist, rows/columns, and initial input; no shell interpolation.
  • Record process group, root PID, start token, provider session ID, executable identity, and auxiliary-process evidence.
  • Termination signals the owned group, waits, escalates, then positively checks survivors before reporting success.
  • PID reuse and escaped auxiliary processes are explicit test cases. Unrelated processes must survive.
  • The broker re-adopts detached hosts by private registry evidence plus the live adoption challenge, peer credentials, and PID/start-token/build readback; it never guesses by process name.
  • On unexpected host or vendor exit, preserve final output, exit status/signal, recent events, and recovery locator as one diagnostic bundle.

18 · Normative v1 contract

The remaining sections are instructions, not aspirations.

Sections 18–28 define terminal foundation v1. If earlier explanatory prose conflicts with these sections, the normative contract wins. A change to a value, state, field, permission, or postcondition below requires a schema/version change and its conformance fixture in the same commit.

Normative language and ownership

  • MUST / MUST NOT are release requirements. SHOULD requires a documented exception. MAY is optional.
  • The Bun daemon is the only hierarchy, task, capability, admission, delivery-policy, and status-fusion authority.
  • A session host is the only owner of one generation’s PTY master, terminal checkpoint, output sequence, and input state.
  • The Workspace is the only owner of native layout, selection, first responder, accessibility presentation, and user-facing terminal preferences.
  • Ghostty is an internal engine. No public Hive record, API, setting key, or error requires Ghostty vocabulary except build/provenance diagnostics.

Common data rules

  • Wire strings are valid UTF-8. Raw PTY and input payloads are byte strings and are never decoded by the transport.
  • IDs are lowercase UUIDv7 text with domain prefixes: run_, agt_, ses_, tsk_, evt_, clm_, txn_, msg_, req_. Legacy IDs are read only during migration.
  • Wall-clock fields are RFC 3339 UTC with millisecond precision. Ordering and deadlines use a monotonic clock; wall time never orders PTY bytes.
  • Revisions and sequence values are unsigned 64-bit integers on the wire. TypeScript rejects values above Number.MAX_SAFE_INTEGER and uses decimal strings in JSON for larger values.
  • outputSeq, checkpointSeq, throughSeq, and renderer high-waters are exclusive byte offsets: every output byte below the value has been applied, and the next frame begins at that value.
  • Unknown identity, authority, process ownership, receipt, or state fails closed. Unknown is preserved and shown; it is never coerced to absent, free, dead, delivered, or allowed.

20 · Local wire protocol v1

Thirty-two bytes frame every request and stream.

Both broker and host endpoints use the same framing. The protocol is local Unix-domain socket only. There is no TCP listener, HTTP upgrade, newline framing, native struct serialization, or implicit machine endianness.

Payload rules

  • Control frames contain one strict UTF-8 JSON object with "schemaVersion":1. Missing/invalid required fields reject the request. Unknown fields are ignored only when the negotiated minor explicitly permits them.
  • OUTPUT, HUMAN_INPUT, AUTOMATION_CHUNK, CREATE_INPUT, and checkpoint byte frames carry raw bytes.
  • Output streamSeq is the first byte offset; the next expected offset is streamSeq + payloadLength. Duplicate bytes are discarded only after digest comparison; a gap forces reattach.
  • There is no compression in wire v1. Checkpoint payloads are engine-owned opaque bytes with their own digest.
  • Parsers read the 32-byte header into fixed storage, validate it, then allocate. They never search the stream for the next magic after a malformed header.

Version and liveness

  • HELLO is the first frame and names client role, build hash, protocol range, instance, and presented grant. No other frame is processed first.
  • WELCOME returns selected version, instance ID, endpoint role, build/engine IDs, connection ID, limits, and monotonic server epoch.
  • Idle connections exchange PING/PONG every 5 seconds. Three missed PONG intervals detach the connection, not the session.
  • Same major/minor may add optional JSON fields only. New frame types, state values, required fields, or changed postconditions require a minor/major bump as appropriate.
  • An unknown frame type with the optional high bit set may be ignored; all other unknown types return UNSUPPORTED_FRAME.

Authentication

  • All endpoints call getpeereid for peer UID/GID and getsockopt(LOCAL_PEERPID) for peer PID before HELLO. They derive the PID start token from proc_pidinfo(PROC_PIDTBSDINFO) start time and the executable with proc_pidpath. JSON identity claims confer no authority.
  • The broker accepts daemon control only when the peer PID/start token matches the current daemon lock and the HELLO values match product, build, protocol, schema, instance, and project from the existing control-daemon handshake.
  • Workspace calls the operator-authorized POST /session-attach-grant. The daemon asks the broker to register a random 256-bit, one-use, 30-second grant hash at the exact host; only the raw token returns over the already-authorized control channel.
  • Raw grants never enter argv, environment, registry files, logs, crash metadata, or analytics. Agents never connect to sessiond; agent status and messages use daemon capabilities.
  • Broker-to-host adoption uses the 32-byte secret passed at host launch over an inherited control file descriptor and retained as adopt.cap. The host also validates peer UID, locator, broker build/protocol, and requested operation.

Local trust boundary

  • Canonicalize HIVE_HOME and every authorized worktree once. Use directory file descriptors, openat, O_NOFOLLOW, ownership checks, and mode checks for runtime access.
  • The v1 boundary prevents accidental cross-project/cross-generation access and stale-token reuse. Same-UID malware is an explicitly accepted residual risk, not a promised sandbox.
  • Terminal bytes, captures, message bodies, clipboard data, and checkpoints are sensitive. Metrics and normal logs contain only IDs, byte counts, ranges, transitions, build IDs, timings, and redacted diagnostic codes.
  • OSC 52 clipboard read and write are denied in v1. HTTP/HTTPS links require an explicit user gesture; file: targets are allowed only inside the canonical repository/worktree after normalization and otherwise require refusal or confirmation.
  • Kitty direct image transmission is supported within §18 limits. File, temporary-file, and shared-memory image transports are disabled.

Workspace event handling

  • The first ordinary text key, deletion, paste, IME composition start, or editing command is held locally while CLAIM_ACQUIRE completes. On denial, Hive keeps the event out of Ghostty and shows the exact owner/recovery action.
  • After ownership, native key, text, preedit, and mouse events go through HiveTerminalKit’s Ghostty encoders. IME owns from composition start through commit/cancel; changing selection does not retarget composition.
  • Return/newline releases with submit; Escape, Control-C, or Control-U release with cancel only when the provider adapter classifies that gesture as draft cancellation. Other navigation/function/mouse gestures are atomic and do not claim.
  • Paste acquires first, uses ghostty_paste_encode against the canonical terminal modes, then sends submit as the next ordered encoded key action. There is no paste sleep, heuristic delay, or 750 ms release timer.
  • A viewer sends monotonically increasing claim-local input sequence values and SHA-256 digests. Reconnect resumes only from the host-acknowledged sequence.

Automated transactions

  • The daemon streams at most 1 MiB into a host buffer. COMMIT must repeat transaction ID, idempotency key, exact length, digest, recipient locator/epoch, durable message ID, provider strategy, and none|return|control-enter submit action.
  • Use libghostty-vt’s public terminal mode, key, mouse, and paste encoders. Bracketed-paste boundaries surround only encoded body bytes; submit is a separate later write. When bracketed paste is off, unsafe control bytes are removed by the encoder and newline conversion follows terminal semantics.
  • Authorization and provider readiness are rechecked before the first committed PTY byte. Cancellation before COMMIT writes nothing. Cancellation after COMMIT returns the stored result; it never attempts compensating keystrokes.
  • Each idempotency key stores digest, locator, epoch, message ID, byte range, write result, and timestamps through session exit. Reuse with different content or scope is MALFORMED_FRAME.
  • Plaintext buffers are fixed-cap, non-core-dump memory where available and are zeroed on commit, cancel, disconnect, and exit. No payload or digest-derived text enters normal logs.

Busy, idle, and priority

  • normal: ordinary guidance at the next proven ready turn boundary. steer: non-destructive guidance at the next supported tool boundary; Claude/Codex use hooks/app-server and Grok honestly degrades to next turn. urgent: cancel the in-flight provider turn, which is never resumed, then inject when input is free; it never deletes/interleaves a human draft or guesses through an unknown modal. critical: immediately and durably revoke write/landing authority; the message intent then decides the process action — pause is a non-destructive SIGSTOP of the same captured process tree, daemon-measured applied when the row is control-paused against this control message and resumed by SIGCONT as the exact same process, while stop/cancel/restrict-writes terminate the old provider generation under lifecycle policy, restart it read-only, and deliver to the new exact generation. Priority and intent are separate axes; these preserve Hive’s current measured semantics.
  • For Claude/Codex terminal delivery, wait up to 5 seconds after injection for a later turn-start proof. Absence does not cause reinjection; state remains injected and shows delayed provider observation.
  • Messages delivered while a provider turn is busy can be transport-injected only if the adapter version explicitly proves queued-input behavior. Otherwise they stay queued until the next safe boundary.
  • Grok never gets a fabricated hook receipt. Transcript activity can reconcile later; stalled injected messages raise typed attention without duplicate writes.
  • The acknowledgment budget begins at injection, not enqueue: urgent 180 seconds, critical 10 seconds. Missing acknowledgment raises typed attention and executes the existing escalation/recovery policy; it does not rewrite provider or agent receipt.

Version support

  • Each adapter manifest pins provider binary version range, launch argv contract, event schema hashes, readiness states, cancel/submit encodings, native-endpoint availability, and fixture set.
  • Launch performs a version/readback probe. An unknown version may run interactively if process containment and terminal fidelity remain safe, but automatic delivery/status features whose evidence changed are disabled and explained.
  • CI runs idle, busy, approval, modal, backpressure, disconnect, restart, and 1 MiB delivery for every supported version. No selector, phrase, or timing guess is patched directly into application UI code.
